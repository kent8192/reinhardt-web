# Migration Guide: 0.1.0 → 0.2.0

Umbrella tracker: [#4520](https://github.com/kent8192/reinhardt-web/issues/4520).
Companion: [#4652](https://github.com/kent8192/reinhardt-web/issues/4652).

## Quick removal index

| Crate | Status |
|---|---|
| reinhardt-core | shipped via PR #4713 |
| reinhardt-query | shipped via PR #4717 |
| reinhardt-di | shipped via PR #4722 |
| reinhardt-conf | shipped via PR #4728 and follow-up RC cleanup |
| reinhardt-db | shipped via PR #4729 |
| reinhardt-auth | shipped via issue #4652 follow-up work |
| reinhardt-rest / -urls / -test / -testkit / -admin | shipped via the 0.2.0 RC cleanup series |

The sections below describe downstream migration from the public 0.1.x
surface to the final 0.2.0 line. They intentionally focus on API changes
visible to application authors, not every internal cleanup commit.

---

## reinhardt-core and reinhardt-urls

### Flat route accessors -> namespaced URL gateways

The old flat URL resolver methods generated by `#[routes]` and `#[viewset]`
were removed. Use the server/client gateway returned by the generated
`ResolvedUrls` value.

```rust
// Before
let detail_url = urls.article_detail(id);

// After
let detail_url = urls.server().blog().article_detail(id);
```

Client route accessors follow the same pattern:

```rust
// Before
let url = urls.blog_client();

// After
let url = urls.client().blog();
```

The deprecated `UrlResolverUnprefixed` path only supported those removed
flat accessors. Prefer the namespaced gateway for new and migrated code.

## reinhardt-query

### `SeaRc<T>` -> `SharedRc<T>`

The transitional `SeaRc<T>` alias from the SeaQuery fork was removed. Use
`SharedRc<T>` directly.

```rust
// Before
use reinhardt_query::SeaRc;
let iden: SeaRc<dyn Iden> = SeaRc::new(MyTable);

// After
use reinhardt_query::SharedRc;
let iden: SharedRc<dyn Iden> = SharedRc::new(MyTable);
```

`SharedRc<T>` keeps the same runtime behavior: it resolves to `Arc<T>` with
the `thread-safe` feature and `Rc<T>` without it.

---

## reinhardt-di

### `Injected<T>` -> `Depends<T>`

The FastAPI-style `Injected<T>` wrapper and `OptionalInjected<T>` alias were
removed. Use `Depends<T>` and `Option<Depends<T>>`.

```rust
// Before
use reinhardt_di::{Injected, OptionalInjected};

#[inject]
db: Injected<Database>,
#[inject]
cache: OptionalInjected<Cache>,

// After
use reinhardt_di::Depends;

#[inject]
db: Depends<Database>,
#[inject]
cache: Option<Depends<Cache>>,
```

The field access model is unchanged: `Depends<T>` dereferences to `T` for
normal handler and service code.

---

## reinhardt-conf

### Monolithic settings -> typed fragments

The deprecated monolithic settings types and magic source helpers were
removed. Build typed settings from explicit sources and fragments.

```rust
// Before
use reinhardt_conf::{auto_source, AdvancedSettings};

let source = auto_source("settings/development.toml");
let settings = AdvancedSettings::from_source(source)?;

// After
use reinhardt_conf::{SettingsBuilder, TomlFileSource};

let settings = SettingsBuilder::new()
    .add_source(TomlFileSource::new("settings/development.toml"))
    .build_composed::<ProjectSettings>()?;
```

TOML is the canonical format in 0.2. `JsonFileSource` and file-extension
auto-detection are no longer part of the public API.

### `TomlFileSource::set_interpolation(bool)` -> builder methods

```rust
// Before
let mut source = TomlFileSource::new("settings.toml");
source.set_interpolation(false);

// After
let source = TomlFileSource::new("settings.toml").without_interpolation();
```

Use `.with_interpolation()` when enabling interpolation explicitly for
readability.

## reinhardt-db

### Re-loading database URL helpers -> composed settings

The deprecated helper that reloaded settings from disk on each call was
removed. Load settings once through the typed settings pipeline, then derive
the database URL from that value.

```rust
// Before
let url = DatabaseConnection::get_database_url_from_env_or_settings(None)?;

// After
let settings = SettingsBuilder::new()
    .add_source(TomlFileSource::new("settings/development.toml"))
    .build_composed::<ProjectSettings>()?;
let url = DatabaseConnection::database_url_from(&settings, None)?;
```

This keeps interpolation, profile selection, and secret handling in one
settings path instead of duplicating file IO inside the database layer.

## reinhardt-auth

### `AuthUser<U>` -> `CurrentUser<U>`

`CurrentUser<U>` is the recommended authenticated-user extractor in the
final 0.2.0 line. `AuthUser<U>` remains as a deprecated compatibility name
and is scheduled for removal in 0.3.

```rust
// Before
async fn handler(AuthUser(user): AuthUser<MyUser>) -> Response {
    let id = user.id();
    // ...
}

// After
async fn handler(CurrentUser(user): CurrentUser<MyUser>) -> Response {
    let id = user.id();
    // ...
}
```

For anonymous-user handling, branch at the framework boundary by returning
401 or redirecting through guards instead of carrying an optional user inside
the extractor.

### `DefaultUser` → `#[user]` macro

`DefaultUser` was removed. Define an application-owned user model.

```rust
// Before
use reinhardt_auth::DefaultUser;

// After
use reinhardt_auth::user;

#[user]
pub struct MyUser {
    pub username: String,
    pub email: String,
    // ...
}
```

### `User` trait + `SimpleUser` + `AnonymousUser` → composable trait stack

The old `core::user` module was removed. Use the composable traits instead:

- `AuthIdentity` for the identity claim
- `BaseUser` / `FullUser` for user model traits
- `PermissionsMixin` for authorization checks

---

## reinhardt-rest

### `OpenApiConfig` -> `OpenApiSettings`

The deprecated ad-hoc OpenAPI configuration struct was removed. Use the
typed settings fragment.

```rust
// Before
let config = OpenApiConfig::default();

// After
let settings = OpenApiSettings::default();
```

When configuration is loaded from TOML, include the `openapi` settings
fragment in the composed project settings type.

---

## reinhardt-test and reinhardt-testkit

### Browser mocks and auth helpers

The old WASM `MockFetch` path was removed. Use the Mock Service Worker
testing surface.

```rust
// Before
let mock = MockFetch::new();

// After
let server = MockServiceWorker::new();
```

The old force-authentication helpers were also removed. Use the fluent auth
test client methods, for example session or JWT login helpers exposed under
`client.auth()`.

### Test user fixtures

The built-in `TestUser` fixture was removed. Define a test-local user type
with `#[user]` and implement the traits needed by the test fixture.

---

## reinhardt-admin and reinhardt-commands

### Deprecated command and asset shims

Deprecated vendor-asset helpers and old command shims were removed during the
0.2.0 RC cleanup. Use the current admin CLI commands and the normal static
asset pipeline instead of calling vendor setup helpers directly.

For project generation, pass the explicit current template/dependency options
accepted by `reinhardt-admin-cli`; removed aliases such as old template-type
flags are not accepted in 0.2.

---

## Migration verification

For application code, run these checks after applying the changes above:

```sh
cargo check --workspace --all-features
cargo test --workspace --all-features
cargo doc --workspace --no-deps --all-features
```

If a downstream crate still references a removed compatibility item, fix the
call site rather than reintroducing the removed symbol. The compatibility
window for RC-deprecated APIs closed with the 0.2.0 line.
