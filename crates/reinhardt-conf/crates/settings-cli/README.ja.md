# reinhardt-settings-cli

設定管理用CLIツール

## 概要

Reinhardt設定ファイルを管理するためのコマンドラインインターフェース。暗号化、バリデーション、環境管理、シークレット処理のためのコマンドをカラー化された出力で提供します。

## 機能

### 実装済み ✓

#### 設定管理

- **Validate**: 構文チェック付き設定ファイルバリデーション
  - TOML、JSON、.envファイルフォーマットサポート
  - プロファイル固有のバリデーション (development, staging, production)
  - .envファイルの行ごとのバリデーション
  - プロファイルベースのバリデーション用`SecurityValidator`との統合

- **Show**: 複数の出力フォーマットで設定値を表示
  - TOML、JSON、.envファイルのサポート
  - ドット記法を使用したネストされたキーのナビゲーション (例: `database.host`)
  - 複数の出力フォーマット: テキスト(カラー化)、JSON、TOML
  - プロファイル対応の設定表示

- **Set**: 設定値の変更
  - ネストされたキーのためのドット記法 (例: `database.port`)
  - 自動型推論 (bool、integer、float、string)
  - `--create`フラグで新しい設定ファイルを作成
  - 変更前の自動バックアップ作成
  - TOML、JSON、.envファイルフォーマットのサポート

- **Diff**: 2つの設定ファイルを比較
  - TOML、JSON、.envファイルの並列比較
  - 追加、削除、変更を表示
  - `--show-values`でオプショナルな値の表示
  - `--only-differences`で差分のみをフィルタ
  - サマリー統計 (総差分数、追加数、削除数)

#### セキュリティ機能

- **Encrypt**: 設定ファイルのAES-256-GCM暗号化
  - 16進数フォーマットの32バイト(256ビット)キーサポート
  - カスタム出力パスまたは自動`.enc`拡張子
  - 暗号化後の元ファイルのオプショナル削除
  - reinhardt-settingsの`ConfigEncryptor`との統合

- **Decrypt**: 暗号化された設定ファイルの復号化
  - AES-256-GCM暗号化ファイルと互換
  - 自動出力パス決定
  - 復号化後の暗号化ファイルのオプショナル削除
  - 16進数キーフォーマットバリデーション

#### 出力ユーティリティ

- **カラー化ターミナル出力**: リッチなターミナル体験
  - 成功メッセージ (緑のチェックマーク)
  - エラーメッセージ (赤の×マーク)
  - 警告メッセージ (黄色の警告シンボル)
  - 情報メッセージ (青の情報シンボル)
  - キーバリューペアのフォーマット
  - テーブルフォーマット機能
  - カラーコード付きの変更を伴うDiff可視化

- **複数の出力フォーマット**:
  - カラー化された構文ハイライト付きテキストフォーマット
  - プリティプリント付きJSONフォーマット
  - プリティプリント付きTOMLフォーマット
