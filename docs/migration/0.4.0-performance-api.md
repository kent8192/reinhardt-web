# 0.4 Performance API Migration

**Applies to:** server-side Reinhardt applications migrating from 0.3.x to
0.4.0.

## What changed

The 0.4 HTTP and routing path removes compatibility allocations from request
construction and dispatch. These changes intentionally alter public request
parameter APIs:

- `Request::query_params` is now a lazy `QueryParams` wrapper instead of an
  eagerly parsed `HashMap<String, String>`.
- Empty `Request::extensions` storage is allocated only after the first
  insertion. `Request::clone_for_di()` still shares an initialized extension
  store, but an empty store is no longer eagerly shared.
- `PathParams::get()` returns `Option<&str>` instead of `Option<&String>`.
- `PathParams::iter()` yields `(&String, &str)` instead of references to owned
  `(String, String)` pairs.
- `PathParams::values()` yields `&str`.
- `PathParams::as_slice()` was removed. Use `PathParams::to_vec()` when owned
  pairs are required.
- `PathParams::insert()` and `Request::set_path_param()` accept borrowed string
  values.

The HTTP/1 adapter also skips body collection for empty `GET` and `HEAD`
requests that have neither `Content-Length` nor `Transfer-Encoding`.

## Query parameters

Most read-only code can keep using `get`, `contains_key`, `len`, `is_empty`,
or `iter`; the first read parses the raw query string.

```rust,ignore
let page = request.query_params.get("page");
```

Code that requires ownership should explicitly materialize a map:

```rust,ignore
let params = request.query_params.to_hash_map();
```

Use `raw_query()` when middleware needs the original query string without
forcing parsed-map allocation:

```rust,ignore
if let Some(raw) = request.query_params.raw_query() {
    trace_query(raw);
}
```

## Path parameters

Code that previously matched on `Option<&String>` should match on `Option<&str>`
or convert at the boundary that needs ownership.

```rust,ignore
// Before 0.4.
let slug: Option<&String> = request.path_params.get("slug");

// 0.4.
let slug: Option<&str> = request.path_params.get("slug");
let owned_slug = request.path_params.get("slug").map(str::to_string);
```

Iteration now borrows values as `&str`:

```rust,ignore
for (name, value) in request.path_params.iter() {
    audit_param(name, value);
}
```

When serialization or compatibility code needs owned pairs, use `to_vec()`:

```rust,ignore
let encoded = serde_urlencoded::to_string(request.path_params.to_vec())?;
```

## Extension storage

Code that only reads extensions continues to use the same API:

```rust,ignore
let user = request.extensions.get::<CurrentUser>();
```

Code that depends on empty cloned requests sharing future insertions must move
that state into an initialized extension before cloning, or pass state through
the DI context. This preserves sharing for real extension state while avoiding
the per-request allocation for handlers that never use extensions.
