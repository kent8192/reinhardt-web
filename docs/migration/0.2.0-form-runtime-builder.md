# Form Runtime Builder Migration

**Applies to:** `reinhardt-pages` form runtime users on 0.2.0 prereleases.

## What changed

`form!` now owns field registration, generated value types, field tokens,
initial values, metadata, validation, loader declarations, and server function
binding. `use_form(&form)` owns dynamic runtime behavior such as dirty/touched
state, errors, dependency reconciliation, reset policy, subscriptions, focus,
and submit lifecycle.

The standalone runtime path based on `FormOptions::new(initial_values)` was
removed. Initial values must come from the generated form definition.

## Before

```rust
use reinhardt_pages::{FormOptions, FormValues, use_form};

#[derive(Clone, PartialEq, FormValues)]
struct ProfileFormValues {
    display_name: String,
}

let runtime = use_form(FormOptions::new(ProfileFormValues {
    display_name: String::new(),
}));
```

## After

```rust
use reinhardt_pages::{form, use_form};

let profile_form = form! {
    name: ProfileForm,
    action: "/profile",
    fields: {
        display_name: CharField {
            initial: String::new(),
            required,
        }
    }
};

let runtime = use_form(&profile_form).build();
runtime.set_value(profile_form.display_name_field(), "Ada".to_string());
```

## Dependency Changes

Use `.deps(...)` on the runtime builder. The default reset policy is
`ResetOnDeps::KeepDirtyValues`: fields changed by the user keep their current
values, while pristine fields receive refreshed defaults.

```rust
let runtime = use_form(&profile_form)
    .deps((user_id, profile_version))
    .build();
```
