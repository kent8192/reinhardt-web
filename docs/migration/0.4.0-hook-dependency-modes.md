# Hook Dependency Modes

Issues [#5511](https://github.com/kent8192/reinhardt-web/issues/5511) and
[#5577](https://github.com/kent8192/reinhardt-web/issues/5577) replace unit and
tuple dependencies with a mandatory named mode.

Issue #5511 established the list-syntax rationale: dependency values should be
named at the call site instead of relying on tuple shape. Issue #5577 expands
that syntax into a breaking, mode-typed API and supersedes the backward-
compatible tuple rollout from #5511.

## Explicit dependencies

Use `deps![...]` when the set of values that should trigger the hook is known
at construction time. `deps![]` is the explicit mount-only form.

```rust,ignore
use_effect(setup, deps![count, filter]);
use_effect(initialize_once, deps![]);
```

Explicit mode subscribes only to the listed reactive values. Reads inside an
effect or memo calculation do not add hidden subscriptions.

## Automatic dependencies

Use `deps_auto!()` for effects, layout effects, and memos when subscriptions
should follow the signals read by the closure.

```rust,ignore
use_effect(setup, deps_auto!());
use_layout_effect(measure, deps_auto!());
let value = use_memo(compute, deps_auto!());
```

Automatic mode records tracked reads during each execution and rebuilds the
subscription set on every setup. Reads performed by cleanup functions are not
tracked. Callbacks, resources, and retained-effect helpers accept only
`deps![...]`, because their callback or future body executes after
construction or is owned by a lifecycle store.

## Replacements

| Removed | Replacement |
| --- | --- |
| `()` | `deps![]` |
| `(count.clone(),)` | `deps![count]` |
| `(count.clone(), filter.clone())` | `deps![count, filter]` |

The second argument is required for every dependency-aware hook. Choose an
explicit list for callbacks, resources, and retained effects; choose either
an explicit list or automatic mode for effects, layout effects, and memos.
