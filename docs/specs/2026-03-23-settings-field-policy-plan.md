# Settings Field Policy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add field-level Required/Optional policy control to the `#[settings]` macro system, with library-defined defaults and user composition-time overrides.

**Architecture:** Extend `SettingsFragment` trait with `field_policies()` metadata method. Fragment macro parses `#[setting(...)]` attributes to generate static policy arrays. Composition macro parses inline `{ field: policy }` override blocks. `SettingsBuilder::build_composed()` validates required fields against the merged `IndexMap` before deserialization.

**Tech Stack:** Rust proc-macro (syn/quote), nom 8.0 parser, serde, indexmap, thiserror

**Spec:** `docs/specs/2026-03-23-settings-field-policy-design.md`

// crates/reinhardt-conf/src/settings/policy.rs

/// Field-level policy for settings fragments.
///
/// Determines whether a field must have an explicit value
/// after all configuration sources are merged by `SettingsBuilder`.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum FieldRequirement {
	/// The field MUST have a value after all sources are resolved.
	/// `SettingsBuilder::build_composed()` returns an error if missing.
	Required,
	/// The field MAY be absent. Default value is used if not provided.
	Optional,
}

/// Metadata describing a single field's policy within a `SettingsFragment`.
///
/// Generated by the `#[settings(fragment = true)]` macro from
/// `#[setting(...)]` field attributes and `default_policy` configuration.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct FieldPolicy {
	/// Field name (matches the struct field name, used for key lookup
	/// in the merged `IndexMap<String, serde_json::Value>`)
	pub name: &'static str,
	/// Whether the field is required or optional
	pub requirement: FieldRequirement,
	/// Whether a compile-time default value is defined via `#[setting(default = "...")]`
	/// or the type implements `Default` (for fields inheriting `default_policy = "optional"`)
	pub has_default: bool,
}
```

	/// Build and validate a composed settings struct.
	///
	/// This method:
	/// 1. Merges all configuration sources
	/// 2. Validates that all required fields have values
	/// 3. Deserializes the merged data into the target type
	///
	/// Fragment-level validation (`validate_fragments()`) should be called
	/// separately by the caller with the appropriate profile.
	pub fn build_composed<T: ComposedSettings>(self) -> Result<T, BuildError> {
		let merged = self.build()?;
		T::validate_requirements(merged.as_map())?;
		let settings: T = merged.into_typed()
			.map_err(BuildError::from)?;
		Ok(settings)
	}

fn policy(input: &str) -> IResult<&str, PolicyKind> {
	alt((
		value(PolicyKind::Required, tag("required")),
		value(PolicyKind::Optional, tag("optional")),
	))(input)
}

fn field_override(input: &str) -> IResult<&str, FieldOverride> {
	let (input, _) = multispace0(input)?;
	let (input, field_name) = ident(input)?;
	let (input, _) = multispace0(input)?;
	let (input, _) = char(':')(input)?;
	let (input, _) = multispace0(input)?;
	let (input, pol) = policy(input)?;
	let (input, _) = multispace0(input)?;
	Ok((input, FieldOverride { field_name, policy: pol }))
}

fn field_overrides(input: &str) -> IResult<&str, Vec<FieldOverride>> {
	let (input, _) = char('{')(input)?;
	let (input, _) = multispace0(input)?;
	// Handle empty block: {}
	if let Ok((input, _)) = char::<&str, nom::error::Error<&str>>('}')(input) {
		return Ok((input, vec![]));
	}
	// Parse comma-separated overrides with mandatory first element
	let (input, first) = field_override(input)?;
	let (input, mut rest) = many0(preceded(
		delimited(multispace0, char(','), multispace0),
		field_override,
	))(input)?;
	let (input, _) = multispace0(input)?;
	let (input, _) = opt(char(','))(input)?; // trailing comma
	let (input, _) = multispace0(input)?;
	let (input, _) = char('}')(input)?;
	rest.insert(0, first);
	Ok((input, rest))
}
```

- [ ] **Step 4: Implement `#[setting(...)]` attribute parsing in `settings_fragment.rs`**

In `crates/reinhardt-core/macros/src/settings_fragment.rs`, add a helper to parse field-level `#[setting(...)]` attributes:

```rust
#[derive(Debug, Default)]
struct FieldSettingAttrs {
	is_required: bool,
	is_optional: bool,
	default_expr: Option<String>,
}

fn parse_field_setting_attrs(field: &syn::Field) -> Result<FieldSettingAttrs, syn::Error> {
	let mut attrs = FieldSettingAttrs::default();
	for attr in &field.attrs {
		if !attr.path().is_ident("setting") {
			continue;
		}
		attr.parse_nested_meta(|meta| {
			if meta.path.is_ident("required") {
				attrs.is_required = true;
				Ok(())
			} else if meta.path.is_ident("optional") {
				attrs.is_optional = true;
				Ok(())
			} else if meta.path.is_ident("default") {
				let value = meta.value()?;
				let lit: syn::LitStr = value.parse()?;
				attrs.default_expr = Some(lit.value());
				Ok(())
			} else {
				Err(meta.error(format!(
					"unknown setting attribute: `{}`. Expected: `required`, `optional`, `default`",
					meta.path.get_ident().map(|i| i.to_string()).unwrap_or_default()
				)))
			}
		})?;
	}

	// Validate: required + default is mutually exclusive
	if attrs.is_required && attrs.default_expr.is_some() {
		let field_name = field.ident.as_ref()
			.map(|i| i.to_string())
			.unwrap_or_else(|| "unnamed".to_string());
		return Err(syn::Error::new_spanned(
			field,
			format!("`required` and `default` are mutually exclusive on field `{field_name}`"),
		));
	}

	// Validate: required + optional is mutually exclusive
	if attrs.is_required && attrs.is_optional {
		return Err(syn::Error::new_spanned(
			field,
			"`required` and `optional` are mutually exclusive",
		));
	}

	Ok(attrs)
}
```

let field_policy_entries: Vec<proc_macro2::TokenStream> = fields.iter().map(|field| {
	let field_name = field.ident.as_ref().unwrap().to_string();
	let attrs = parse_field_setting_attrs(field)?;

```rust
fn validate_overrides(
	type_ident: &syn::Ident,
	fields: &syn::Fields,
	overrides: &[FieldOverride],
) -> Result<(), syn::Error> {
	let field_names: Vec<String> = match fields {
		syn::Fields::Named(named) => named.named.iter()
			.filter_map(|f| f.ident.as_ref().map(|i| i.to_string()))
			.collect(),
		_ => vec![],
	};

	// Check for duplicate overrides
	let mut seen = std::collections::HashSet::new();
	for ov in overrides {
		if !seen.insert(&ov.field_name) {
			return Err(syn::Error::new(
				proc_macro2::Span::call_site(),
				format!("duplicate override for field `{}` in `{}`", ov.field_name, type_ident),
			));
		}
	}

	// Check field existence
	for ov in overrides {
		if !field_names.contains(&ov.field_name) {
			let suggestion = find_closest_match(&ov.field_name, &field_names);
			let mut msg = format!("field `{}` does not exist in `{}`", ov.field_name, type_ident);
			if let Some(s) = suggestion {
				msg.push_str(&format!("\n  did you mean `{s}`?"));
			}
			return Err(syn::Error::new(
				proc_macro2::Span::call_site(),
				msg,
			));
		}
	}

	Ok(())
}

fn find_closest_match(target: &str, candidates: &[String]) -> Option<String> {
	candidates.iter()
		.map(|c| (c, levenshtein_distance(target, c)))
		.filter(|(_, d)| *d <= 3.min(target.len() / 2))
		.min_by_key(|(_, d)| *d)
		.map(|(c, _)| c.clone())
}

fn levenshtein_distance(a: &str, b: &str) -> usize {
	let a_len = a.len();
	let b_len = b.len();
	let mut matrix = vec![vec![0; b_len + 1]; a_len + 1];
	for i in 0..=a_len { matrix[i][0] = i; }
	for j in 0..=b_len { matrix[0][j] = j; }
	for (i, ca) in a.chars().enumerate() {
		for (j, cb) in b.chars().enumerate() {
			let cost = if ca == cb { 0 } else { 1 };
			matrix[i + 1][j + 1] = (matrix[i][j + 1] + 1)
				.min(matrix[i + 1][j] + 1)
				.min(matrix[i][j] + cost);
		}
	}
	matrix[a_len][b_len]
}
```

**Field Validation Strategy:**

Since the proc-macro cannot inspect external types' fields via `syn`, use a two-pronged approach:

1. **Duplicate detection**: Check within the override list itself (no external type needed). This is done in the `validate_overrides` function above using a `HashSet`.

2. **Field existence**: Generate a const block that references each overridden field by name on the target type. If the field doesn't exist, the Rust compiler produces an error naturally. For better error messages, also generate a `compile_error!` using `field_policies()`:

```rust
// Generated code for override validation:
const _: () = {
	// Verify that override field names match actual field_policies() entries
	let policies = <CoreSettings as SettingsFragment>::field_policies();
	// At compile time, we can't iterate const slices easily.
	// Instead, generate field access expressions:
	fn _assert_field_exists(s: &CoreSettings) {
		let _ = &s.debug;          // compiler error if `debug` doesn't exist
		let _ = &s.allowed_hosts;  // compiler error if `allowed_hosts` doesn't exist
	}
};
```

This gives immediate compiler errors with the field name in the message. The `validate_overrides` function with `syn::Fields` is used only when the type definition IS available in the same compilation unit (e.g., for system fragments defined alongside the compose macro).

## Implementation Notes

### Dependency Graph (Resolved)

The macro crate (`reinhardt-core/macros`) generates code that references `reinhardt-conf` types via fully-qualified paths like `#conf_crate::settings::policy::FieldRequirement`. This works because the generated code runs in the user's crate which depends on `reinhardt-conf`. The macro crate itself does NOT depend on `reinhardt-conf`.

### `PolicyKind` in Macro Crate (Resolved)

The parser defines a local `PolicyKind` enum (Required/Optional) for internal representation. During code generation, `PolicyKind` values are converted to `FieldRequirement` token references in the output.

### Compile-Time Field Validation (Resolved)

The composition macro generates a const assertion block that accesses overridden fields by name on the target type. If a field doesn't exist, the Rust compiler produces a natural error. Duplicate override detection is done within the macro itself using a `HashSet`.

### Flat Key Lookup and `#[serde(flatten)]` (Critical Assumption)

The composition macro applies `#[serde(flatten)]` to ALL fragment fields. This means ALL fragment fields are flattened into the top-level namespace of the merged `IndexMap`. Therefore `validate_requirements` looks up field names as top-level keys (e.g., `"debug"`, `"allow_origins"`) — NOT section-prefixed keys (e.g., `"core.debug"`, `"cors.allow_origins"`).

This is consistent with how the existing deserialization works: serde's flatten already expects top-level keys. The `section()` value is used only for:
1. TOML section headers during source loading (before merging)
2. Error messages in `MissingRequiredField`

If a future change removes `#[serde(flatten)]`, the key lookup strategy in `validate_requirements` must be updated accordingly.

### `validate_fragments` vs `validate` Naming (Deliberate Deviation from Spec)

The `ComposedSettings` trait uses `validate_fragments()` instead of the spec's `validate()` to avoid confusion with `SettingsFragment::validate()`. This is a deliberate improvement.
