# Client Form Bindings Implementation Plan

**Goal:** Add opt-in DTO-derived client form bindings for issue #5550, including
typed form runtime integration, DTO assembly, enum choices, validation, and an
optional server function submit helper.

**Architecture:** `#[derive(ClientForm)]` lives in `reinhardt-pages-macros` and
emits `{Dto}ClientForm` companion types that implement `FormRuntimeSource`.
`reinhardt-pages` owns small generated-code support traits and a generic async
submit primitive. Enum choice metadata is generated by a separate
`#[derive(ClientFormChoices)]` because a struct derive cannot inspect external
enum variants.

**Tech Stack:** Rust 2024, `syn`, `quote`, `proc_macro_crate`, `trybuild`,
existing `reinhardt-pages` form runtime tests.
