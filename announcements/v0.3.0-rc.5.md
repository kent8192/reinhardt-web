# reinhardt-web v0.3.0-rc.5

## Highlights

This release delivers a broad set of request hot-path performance improvements across the routing, HTTP, and server-function layers. `ServerRouter` now borrows already-normalized paths and avoids alternate-path allocations, skips cookie-jar setup for body-only server functions, and deserializes JSON request bodies directly from bytes. `PathParams` stores small parameter sets inline using `smallvec`, and single-middleware chains avoid one composed-handler heap allocation per request. Taken together, benchmarks show 19–33% latency reductions for plain GET, path-parameter, and JSON server-function endpoints, and 38–67% allocation reductions across common dispatch scenarios.

Database performance sees significant gains as well. The DB connection pool now skips event bookkeeping entirely when no listeners are registered and generates connection IDs lazily, cutting listener-free acquire/release overhead by 44% and halving the allocation count to match raw sqlx. Migration graph topological sorting replaces repeated full-graph scans with a precomputed adjacency map, reducing plan-generation time by 99.2% on large graphs (10,000 migrations: 777 ms → 6 ms). Admin list views now combine the row fetch and filtered total count into a single `COUNT(*) OVER()` query, halving DB round-trips for non-empty pages.

Browser WASM and hot-reload latency also improve in this release. The Pages browser-WASM check path replaces the `reqwest` dependency with an internal Fetch API wrapper, trimming 21 dependency nodes and reducing browser-WASM check time by 23.6%. The `runserver` autoreload debounce window drops from 300 ms to 120 ms, cutting the fixed watcher wait by 60% and reducing the static-page hot-patch browser-visible tail from 305 ms to 125 ms. Release-profile WASM binaries for the Pages SPA fixtures shrink by 26% before any `wasm-opt` post-processing.

Two notable bug fixes land in this release. The Pages quickstart scaffold now correctly defaults new projects to SQLite in the non-interactive path, making `manage check`, `manage migrate`, and `manage runserver` work out of the box without manual configuration. CockroachDB migration execution now holds the sentinel-row serialization lock across the full apply, rollback, plan, and single-migration window, preventing concurrent migrators from running migration bodies simultaneously after schema-table bootstrap.

## Breaking Changes

No breaking changes in this release.
