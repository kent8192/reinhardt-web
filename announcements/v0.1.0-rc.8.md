# reinhardt-web v0.1.0-rc.8

## Highlights

This release focuses on correctness fixes in the project scaffolding commands and a feature flag configuration bug.

The `startproject` and `startapp` code generation commands receive several important fixes. Generated `Cargo.toml` files now use the correct `{{ reinhardt_version }}` template variable instead of a literal `*` wildcard, and app `lib.rs` templates now include the missing `use reinhardt::app_config;` import — both of which previously caused compilation errors when following the tutorials. Additionally, the generated settings template now provides a `base_dir` default sourced from `CARGO_MANIFEST_DIR`, preventing `test_get_settings` from panicking due to a missing required field. The `startapp` command now outputs the app entry point as `src/apps/{name}.rs` rather than `src/apps/{name}/lib.rs`, correctly following Rust 2024 Edition module path conventions.

The `full` feature flag is corrected to include `client-router`. Users who specify `features = ["full"]` in their `Cargo.toml` now have access to `UnifiedRouter` as expected, instead of finding it silently absent.

A Clippy lint (`collapsible_if`) in `start_commands` is resolved by collapsing nested `if` guards into a single condition using Rust 2024 let-chains, keeping the build clean under `-D warnings`. CI guard workflows are also fixed to avoid spurious cancellations triggered by non-`migration-approved` label events.

## Breaking Changes

No breaking changes in this release.
