# reinhardt-web v0.3.0-rc.1

## Highlights

This release candidate lands the 0.3.0 feature set in full. The biggest theme is **WASM/native API parity**: a formal P0/P1/P2 contract now governs every public surface. The new `#[client_page]` macro emits native route-table stubs so client URL patterns can be named cfg-free in shared code, and `#[injectable]`/`#[injectable_factory]` gain WASM no-op stubs so provider functions compile without call-site `#[cfg]` guards. Native-only admin and DI registration behavior is gated from WASM output automatically.

The **dependency injection system** is redesigned around explicit key types. `InjectableKey`, `FactoryOutput<K, T>`, and `Depends<K, T>` replace value-type-only provider identity, eliminating the need for wrapper types to avoid registry collisions. `#[injectable]` is now the provider-function macro; `#[injectable_factory]` is retained as a deprecated alias. The `DependsResult` and `DependsOption` sugar aliases are removed in favour of `Depends<K, Result<T, E>>`.

Pages gains a cluster of new primitives: the `#[wasm_server_api]` macro enforces matching public signatures across WASM and server targets; `#[component]` adds route-backed component macros with `Path`/`Query` extractors; `Portal`/`mount_portal` provides explicit RAII portal mounting; `ActivityBoundary` and `ViewTransitionBoundary` cover state-preserved hidden regions and View Transition API coordination; and the `form!` runtime adds file-field support, dynamic `FieldArray` collections, an experimental `CustomWidget` adapter, and an expanded native widget matrix. Hot-reload latency is substantially reduced, with a compile-free static `page!` hot-patch path for pure UI edits.

On the ORM side, generated `{Model}Info` relation fields now use lightweight `RelationInfo<T>` and `ManyToManyInfo<Source, Target>` payloads instead of flattened `*_id` scalars, and `makemigrations` detects unambiguous field renames and emits safe `RenameColumn` operations instead of destructive add/drop pairs. The formatter now runs rustfmt inside `page!` expression islands, and `fmt-all` correctly skips nested Cargo workspaces to resolve the long-standing `fmt` vs `fmt-all` canonical-form conflict. All deprecated 0.2 compatibility APIs (`AuthUser`, `create_resource`, `use_effect_event`) are removed; see the migration guide for replacements.
