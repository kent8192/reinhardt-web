# reinhardt-web v0.1.0-alpha.3

## Highlights

This release significantly expands the documentation available to new and migrating users. Six new cookbook guides now cover common real-world tasks including CORS configuration, custom middleware creation, pagination patterns, request body parsing, response serialization, and static file serving. Alongside these, dedicated migration guides walk developers through moving existing applications from Actix-web, Axum, and Django to reinhardt-web. A new troubleshooting section addresses common errors and performance optimization strategies. Several example projects (github-issues and twitter) are also updated with production-ready patterns such as structured error responses, paginated response wrappers, and middleware stacks.

The project's README is now available in seven additional languages — Japanese, Simplified Chinese, Traditional Chinese, Russian, Ukrainian, Persian, and Arabic — making the framework more accessible to a global audience.

On the tooling side, release automation migrates from cargo-workspaces to [release-plz](https://release-plz.dev/), which brings automatic version bumping driven by Conventional Commits, automatic CHANGELOG generation via git-cliff, and automatic Release PR creation. The Version Cascade Policy previously maintained manually is no longer needed and has been removed. The CI pipeline also gains Docker image caching to prevent hitting Docker Hub rate limits, along with disk-space management improvements across multiple workflows.

## Breaking Changes

No breaking changes in this release.
