# reinhardt-web v0.1.0-rc.22

## Highlights

The `form!` macro gains a first-class `strip_arguments` mechanism that lets you explicitly route arbitrary expressions — most commonly a CSRF token — to the underlying `server_fn` call. Previously, the macro silently appended a hidden `__csrf_token: String` argument to every non-GET server function, which caused arity mismatches at the WASM compilation boundary when a server function did not declare that parameter. With `strip_arguments`, the wiring is visible at the call site, the compiler validates it, and the mechanism is not CSRF-specific. The validator also rejects duplicate keys and collisions with declared form fields at compile time. Backward-compatible auto-injection remains in place for unmigrated forms; see PR #3972 for a migration guide.

The scaffold generated by `reinhardt-admin startapp --with-pages` and `startproject --with-pages` now matches the layout used by the Basis tutorial and `examples/examples-tutorial-basis/`. Previously, following the tutorial against a freshly-generated project hit compile errors at Part 3 because the scaffold placed `server_fn` under `src/server/` while the tutorial imports it via `crate::server_fn::`. The corrected scaffold puts `server_fn` at the right path, adds `src/config/wasm.rs`, and includes the `mod server_only` re-export shim, so the generated project compiles without manual restructuring.

Quick-start and admin-CLI documentation now includes an explicit `--version "0.1.0-rc.22"` flag for `cargo install reinhardt-admin-cli`. During the RC phase Cargo does not select pre-release versions by default, so the previous install snippet returned a registry error for new users.

## Breaking Changes

No breaking changes in this release.
