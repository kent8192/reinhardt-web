# reinhardt-web v0.1.0-alpha.11

## Highlights

This release delivers targeted correctness fixes to the `reinhardt-query` query builder and the database layer, along with validation improvements and internal code-quality work.

The `Iden` derive macro receives three correctness fixes: it now correctly handles the `Table` enum variant for identifier derivation, parses `Meta::List`-style iden attributes, and reads the `iden` attribute from the struct level rather than the first field. Together these changes make the macro behave predictably across a much wider range of use cases, and a missing `DropBehavior` import in revoke statement tests is also restored.

A notable data-integrity bug in the database layer is resolved: insert values in the many-to-many relationship manager were silently discarded instead of being bound to the query. This fix ensures that many-to-many insert operations correctly persist all supplied values to the database.

Input validation is tightened in the `CreateUser` and `GrantRole` query builders, which now reject whitespace-only name strings. This prevents malformed queries from being generated at the source rather than failing at the database level.

On the infrastructure side, two partial release failures are diagnosed and recovered from: unused dev-dependencies that caused `cargo publish` resolution errors are removed, and the release tooling is updated to set `release_always = true` so that all version-bumped crates are consistently published, preventing future phantom-skip failures.

## Breaking Changes

No breaking changes in this release.
