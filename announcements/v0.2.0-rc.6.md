# reinhardt-web v0.2.0-rc.6

## Highlights

This release introduces the new `reinhardt-providers` workspace crate, which houses narrow cloud-provider integrations. The initial surface covers AWS S3: `PUT`, `GET`, `DELETE`, `HEAD`, and presigned `GET` URL operations are implemented over a lightweight `reqwest`/SigV4 path, replacing the previously direct `aws-sdk-s3` dependency in `reinhardt-storages`. The AWS SDK default provider chain for credentials and region resolution (profiles, IMDS, ECS task roles, IRSA, and similar sources) is preserved through `aws-config`, so existing credential configuration continues to work unchanged.

The `form!` macro's WASM codegen fixes a compiler warning emitted when a form declares no fields. The generated `runtime_set_focus` implementation previously matched an uninhabited field enum and then emitted unreachable DOM focus code beneath it. The macro now emits the exhaustive `match field {}` expression as the complete focus body, eliminating the warning without altering the runtime behavior of forms that do have fields.

The tutorial example crates have been substantially overhauled. The REST example's function-based handlers now use DI-provided real database access instead of in-memory mock data, and a new teaching endpoint demonstrates `Depends<Result<T, E>>` for fallible factory registration. The basis example now registers server functions through per-app `urls` modules and replaces manually maintained client link helpers with a unified `reverse()` function for type-safe client-side URL generation.

The 0.1.x → 0.2.0 migration guide is now complete, with dedicated sections for routing, authentication, ORM, forms, settings, and database migrations, plus a verification checklist and quick-scan commands. Website tutorials have been synchronized with the current example crate structure, and installation snippets across READMEs and docs.rs entries now reference the stable 0.2.0 line.

## Breaking Changes

No breaking changes in this release.
