# reinhardt-web v0.4.0-alpha.2

## Highlights

This release focuses on polishing the Pages macro system and reactive form/i18n APIs. The `page!` macro no longer misinterprets named `t!` interpolation keys (e.g. `id = project_id`) as free variables, and generated `bind:` control lowering now passes `Copy` signal handles by value instead of emitting redundant clones — both fixes remove spurious Clippy failures and correctness edge cases in generated code. Generated Pages apps also gain a standard `client/hooks.rs` module as the canonical home for reusable custom hooks.

Internationalization gets several improvements: `makemessages` now correctly discovers and updates per-app locale catalogs registered via `register_app_locale!` instead of writing only the root catalog, and its extraction now parses `t!` calls with `syn` for more reliable interpolation handling. `ClientLauncher` gains `provide_context` and a feature-gated `i18n_context` helper so fresh CSR applications can retain typed root contexts (including i18n catalogs) across navigations with proper RAII cleanup, and the Pages facade now re-exports the core i18n catalog and translation APIs directly.

On the server side, `#[server_fn(pre_validate = true)]` now validates deserialized DTO arguments directly, fixing endpoints that previously failed to compile. `runserver --with-pages` now correctly serves collected static assets under the configured `STATIC_URL` instead of falling back to the SPA HTML shell. New `FormActionButton` and `FormActionResultPanel` components add headless submit/result UI backed by validated form actions and resource reconciliation.

Notably, this release also includes a **breaking change**: the public ORM `DatabaseConnection` is replaced by a generational, `Copy`-safe handle paired with a new `DatabaseConnectionLease` for RAII-backed ownership — see the migration guide referenced in PR #5791. Dependency injection macros also now correctly support `mut` and tuple-struct destructuring patterns across HTTP, GraphQL, gRPC, and WebSocket handlers.

## Breaking Changes

No breaking changes in this release.
