# reinhardt-web v0.1.1

## Highlights

This patch release closes several runtime-correctness gaps discovered shortly after the 0.1.0 stable launch. The most critical fix resolves a dependency-injection bug in `#[injectable_factory]`: non-`Depends` `#[inject]` parameters were resolved exclusively via the global registry, causing a `DependencyNotRegistered` panic that the `#[server_fn]` layer masked as a generic HTTP 500. Authentication errors that should have surfaced as 401/403 responses were silently swallowed. The fix adds a registry-first fallback to `Injectable::inject` for non-`Depends` parameters, matching the resolution behaviour already used by `#[server_fn]` and `#[routes]`.

A second significant fix addresses the `runserver` development server: routes registered via `#[get]`, `#[post]`, and `#[url_patterns]` were correctly visible in the OpenAPI explorer but were never actually dispatched — `handle_request` only consulted the static-file and SPA fallback paths. Router dispatch is now inserted before those fallbacks, with 404/405 responses continuing to fall through as before. Additionally, projects scaffolded with `reinhardt-admin startproject --with-rest` failed to compile because `#[routes]` attempted to resolve WebSocket resolver modules that REST-only projects do not provide; the generated URL template now passes `server_only` and `no_ws_resolvers` flags to suppress that resolution.

The `reinhardt-admin-cli` formatter gains support for `form!` macro DSL blocks, mirroring the existing `page!` pipeline. Blank lines and comments inside both `page!` and `form!` DSL blocks are now preserved during formatting and codemod rewrites. A new `migrate-manouche-v2` subcommand scaffolds an automated v1→v2 codemod pipeline with a walker, a pluggable rule trait, and initial migration rules.
