# reinhardt-web v0.1.0-rc.25

## Highlights

The `reinhardt-commands` crate now exposes `auto_register_router()` as a public symbol and introduces a new `start_server(addr: &str)` convenience helper (gated on `feature = "server"`). Previously, container-style binaries that wanted to skip the `manage` CLI surface had no clean path to start the HTTP server — callers were forced to synthesize fake argv or duplicate the route-registration logic internally. `start_server` handles route auto-registration and delegates to `RunServerCommand` in one call, while the newly public `auto_register_router` is available for callers that need finer-grained composition with `RunServerCommand::execute()` directly.

`Signal<T>` is now `Send + Sync` on native targets. The inner storage switches from `Rc<RefCell<T>>` to `Arc<RwLock<T>>` on non-`wasm32` platforms, unblocking dependency-injection scenarios where a router newtype wrapping `UnifiedRouter` (through `ClientRouter`) previously failed to compile with an `E0277` thread-safety error. The `Rc<RefCell<T>>` path is preserved on `wasm32` to keep the reactive hot path zero-cost in SPA contexts.

Database URL validation now occurs at `ApplicationBuilder::build()` time. Previously, unrecognized or malformed URL schemes were accepted silently and only surfaced as opaque connection failures at runtime. The fix raises a `BuildError::DatabaseError` immediately on `build()`, reusing the scheme allowlist (`postgres`, `postgresql`, `sqlite`, `mysql`, `mariadb`) already enforced in `reinhardt-conf` — failing strictly earlier than connection time without altering the `ApplicationDatabaseConfig::new()` constructor signature.

## Breaking Changes

No breaking changes in this release.
