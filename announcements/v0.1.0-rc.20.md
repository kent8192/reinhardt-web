# reinhardt-web v0.1.0-rc.20

## Highlights

This release focuses on correctness: a critical scaffolding regression is fixed and a comprehensive documentation accuracy sweep is applied across the entire crate ecosystem.

**Scaffolding bug fix:** The `reinhardt-admin startproject --with-pages` command was broken in rc.19 â€” every generated project immediately failed to build because `default-run` was set to the project name while the template's only binary target is `manage`. This affected 100% of `--with-pages` scaffolds, including the blog demo walkthrough. The fix aligns `default-run = "manage"` with the actual binary and adds a manifest-parsing integration test to prevent the same class of regression from slipping through CI in the future.

**Documentation accuracy overhaul:** A systematic audit of crate READMEs uncovered widespread API inaccuracies, many of which would produce compile errors if followed verbatim. Key corrections include: exported type names in `reinhardt-auth` (`InMemoryTokenBlacklist`, `InMemoryRefreshTokenStore`); the actual `ConnectionPool` API in `reinhardt-db` (`new_postgres()` constructor and `.acquire()` instead of the non-existent builder chain and `.get()`); removal of entirely fabricated types from `reinhardt-admin` (`AdminHandlers`, `AdminRouter`, drag-and-drop structs); the correct JavaScript engine name in `reinhardt-dentdelion` (`boa_engine`, not `deno_core`); and import paths, feature flag names, middleware type names, and required struct fields across `reinhardt-http`, `reinhardt-graphql`, `reinhardt-grpc`, `reinhardt-conf`, `reinhardt-commands`, `reinhardt-mail`, and `reinhardt-core`. The root `README.md` is updated throughout to use current patterns â€” `#[settings]`, `#[user]`, `AuthUser<U>` extractor, `Query<T>`, and ORM filter syntax.

**Version-reference tooling:** A new marker-based `update-version-refs.sh` script and `validate-version-markers.sh` CI lint lay the groundwork for automatically updating hardcoded version references in documentation files on each release, eliminating the post-release follow-up PR pattern.

## Breaking Changes

No breaking changes in this release.
