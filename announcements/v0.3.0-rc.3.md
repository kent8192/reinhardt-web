# reinhardt-web v0.3.0-rc.3

## Highlights

The `#[user]` proc-macro is now inert on `wasm32-unknown-unknown`. Shared models that carry `#[user]` can now compile for WASM without any auth runtime behavior leaking into the client bundle. Native builds continue to receive full auth integrations, manager code, and M2M injections as before. This eliminates the need for `cfg_attr(native, user(...))` workarounds in shared model files.

Pages app scaffolding now generates a target-neutral layout from the start. The server-only implementation surfaces move under a `server/` submodule gated with `#[cfg(server)]`, while `pages.rs`, `urls.rs`, and `server_fn.rs` remain accessible to both native and WASM compilation targets. Freshly generated apps pass native checks, WASM checks, and `showurls` without requiring manual repairs. This release also resolves several live tutorial follow-ups: migration drift from no-op FK `AlterColumn` rounds is suppressed, OrbStack/TestContainers gets a `DOCKER_HOST` fallback, and the UnoCSS reset SRI hash is updated to match the pinned asset.

A batch of viewset serializer fixes ensures that `ModelViewSet` and `ReadOnlyModelViewSet` usages throughout tests, doctests, and the REST tutorial example are wired with concrete `JsonSerializer<T>` implementations rather than empty stubs or `()` placeholders. These corrections unblock CI and align the examples with the current `Serializer + Default` type-bound contract.

## Breaking Changes

No breaking changes in this release.
