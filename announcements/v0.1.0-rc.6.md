# reinhardt-web v0.1.0-rc.6

## Highlights

Two critical security vulnerabilities are patched in this release. A SQL injection flaw in `reinhardt-db`'s PostgreSQL filter operators allowed malicious field names passed to `filter.field` to break out of identifier context via `format!()` string interpolation. The fix introduces a `quote_identifier` helper and migrates `OuterRef` comparisons to SeaQuery's type-safe API across 18 affected locations. Separately, the JWT and MFA authentication backends no longer return fabricated user data — fake `@example.com` email addresses and non-deterministic UUIDs are replaced with honest empty defaults and deterministic `Uuid::new_v5()` generation.

`Depends<T>` in the DI layer now reaches feature parity with `Injected<T>`. Circular dependencies previously caused infinite recursion and stack overflows; the container now detects cycles and returns an error. The type also gains `AsRef<T>`, `as_arc()`, and `metadata()` methods, and its raw `use_cache` bool is replaced with a structured `InjectionMetadata`.

A WASM compilation regression that caused `reinhardt-web@0.1.0-rc.5` to be yanked from crates.io has been corrected. A missing `#[cfg(not(target_arch = "wasm32"))]` gate on the `reinhardt_macros` re-export prevented the crate from building for `wasm32-unknown-unknown`. CI now verifies WASM compilation across multiple crates (`reinhardt-web`, `reinhardt-pages`, `reinhardt-query`) and feature combinations on every PR.

`reinhardt-admin fmt` now runs rustfmt on Rust expression blocks inside `page!` macro DSL — complex expressions exceeding 100 characters are reformatted into properly indented multiline blocks instead of remaining as unwieldy single lines. The migration code generator also fixes serialization of the `initial` field, resolving assertion failures in `makemigrations` E2E tests.

Finally, `warn(missing_docs)` is now enforced across the entire workspace. Over 2,000 public API items have been documented, with reinhardt-db (1,282 items), reinhardt-core (376), reinhardt-forms (355), reinhardt-rest (180), and reinhardt-utils (147) receiving the most extensive coverage.

## Breaking Changes

No breaking changes in this release.
