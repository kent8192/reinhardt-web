# reinhardt-web v0.1.0-rc.2

## Highlights

This release focuses on security hardening and bug fixes accumulated since `0.1.0-rc.1`.

Multiple SQL injection vulnerabilities are patched across `BatchUpdateBuilder`, `QuerySet` filter operators (JSONB, FullTextMatch, Array, Range), `PolymorphicQuery`, and `UpdateBuilder` WHERE clauses — all replaced with parameterized queries. The HTTPS redirect middleware now validates the `HOST` header against a configurable allowlist, closing a host header injection vector. Flatpages HTML-escapes title and content output to prevent stored XSS. The OAuth2 provider validates that `client_id` matches the issued authorization code (preventing client confusion attacks), enforces a 10-minute code TTL per RFC 6749, and `ObjectPermission`/`RoleBasedPermission` are corrected to check actual object-level grants and role assignments instead of only confirming that a user is authenticated.

Several concurrency bugs and unsoundness issues are resolved. A use-after-unlock unsoundness in `LazyLoaded::get()` is eliminated by returning owned values. Three race conditions in `reinhardt-tasks` are fixed: Redis `update_status` uses an atomic Lua script, worker locks are released unconditionally even when `store_result` fails, and `TaskLock::extend()` is made atomic. `DynamicSettings` replaces `parking_lot::Mutex` with `tokio::sync::Mutex` to prevent deadlocks across `.await` points. Rate limit token consumption is now atomic to prevent concurrent requests from bypassing limits. `SignalRegistry::get_or_create` uses an upgradable read lock to eliminate a TOCTOU race.

WebSocket reliability improves in several ways: connection slots are correctly released on disconnect in `RateLimitMiddleware`, `Room::send_to` releases its lock before sending to avoid blocking join/leave operations, `HeartbeatMonitor` uses `tokio::select!` for early pong detection, and stale IP entries are pruned from `ConnectionRateLimiter` to prevent unbounded memory growth.

Other notable fixes include the HTTPS redirect status code changing from 301 to 308 (preserving HTTP method and body), `makemigrations` correctly mapping `auto_increment`, `not_null`, and `DateTime<Utc>` field attributes, `CharField` validation using character count instead of byte length for correct multi-byte Unicode handling, Hungarian plural forms being correctly categorised in `reinhardt-i18n`, and the `Priority` enum in `reinhardt-tasks` ordering by weight value so `Custom(n)` ranks correctly relative to `Low`, `Normal`, and `High`.

## Breaking Changes

No breaking changes in this release.
