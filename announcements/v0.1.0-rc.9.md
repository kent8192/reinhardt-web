# reinhardt-web v0.1.0-rc.9

## Highlights

The `reinhardt-web` facade gains significant API surface in this release. `reinhardt::query` is now accessible under the `database` feature, and `reinhardt::graphql`, `reinhardt::i18n`, `reinhardt::mail`, `reinhardt::grpc`, `reinhardt::dispatch`, and `reinhardt::deeplink` are exposed under their respective feature flags. Users who depend on the single `reinhardt` crate can now reach these subsystems without declaring separate crate dependencies.

Several security issues are addressed. SQL injection vulnerabilities in `AttachDatabaseStatement`, `DetachDatabaseStatement`, and PostgreSQL extension and schema management operations are fixed by properly escaping identifiers and string values. Three XSS vectors are patched: `ScriptTag::inline` content no longer allows premature tag closure, form widget attribute keys are now HTML-escaped alongside values, and the SSR `lang` attribute escapes embedded double quotes. In the auth crate, `SessionConfig` now defaults `cookie_secure` to `true`, token comparisons use constant-time equality to prevent timing attacks, and OAuth2/OIDC flows enforce HTTPS for all endpoint URLs (with an exemption for loopback addresses during local development). Database connection strings are redacted from `Debug` output to prevent accidental credential exposure in logs.

`LeakyBucketThrottle` now correctly isolates rate-limit state per key rather than sharing a single bucket across all clients, aligning its behavior with `TokenBucketThrottle`. Both throttle backends add bounded HashMap growth (defaulting to 10,000 entries) with two-phase eviction — idle-entry removal first, LRU fallback second — to prevent unbounded memory consumption under high-cardinality key workloads. Token bucket state initialization switches to lazy evaluation to avoid unnecessary clock reads on existing keys.

Project scaffolding generated by `startproject` and `startapp` receives several correctness fixes: RESTful app templates now use `ServerRouter` instead of `UnifiedRouter`, the required `ctor` crate is included in both template variants so `#[model(...)]`-annotated structs compile out of the box, and the `type=mtv` option is correctly respected during template selection.

A broad panic-prevention sweep across `reinhardt-db`, `reinhardt-http`, `reinhardt-views`, `reinhardt-graphql`, `reinhardt-middleware`, `reinhardt-urls`, `reinhardt-utils`, `reinhardt-pages`, `reinhardt-rest`, and `reinhardt-core` replaces `.unwrap()` calls on mutex and RwLock guards with the poison-recovery pattern (`unwrap_or_else(|e| e.into_inner())`), preventing cascading failures when a thread panics while holding a lock. `get_websocket_router()` is also fixed to return the globally registered router rather than a fresh empty one.

## Breaking Changes

No breaking changes in this release.
