# reinhardt-web v0.1.0-rc.16

## Highlights

The admin panel ships as a fully production-ready WASM SPA this release. Compiling, bundling, and serving WASM now happens automatically—vendor CSS and fonts are downloaded during `collectstatic`, UnoCSS utility classes are generated, and `runserver --with-pages` triggers the WASM build with no extra steps. The login flow uses JWT tokens stored in HTTP-Only, SameSite=Strict cookies with an Origin Guard for CSRF defense. Staff status is correctly embedded in JWT claims and propagated through `JwtAuthMiddleware`. The new `AdminSettings` fragment makes the admin panel configurable via TOML: custom CSP directives, security headers, site title, login/logout URLs, and pagination are all supported.

The dependency injection API is significantly streamlined. `#[inject]` parameters now use `Depends<T>` instead of `Arc<T>` across all macros—`#[get]`, `#[post]`, `#[server_fn]`, `#[routes]`, and `#[injectable_factory]`. `Injected<T>` and `OptionalInjected<T>` are deprecated in favor of `Depends<T>` and `Option<Depends<T>>`. The `#[injectable]` macro no longer auto-derives `Clone`, supporting non-clonable types in DI. A pseudo orphan rule prevents accidentally overriding framework-managed types. The new `Validated<T>` extractor wrapper auto-calls `.validate()` at injection time. Factory-registered types now resolve correctly via `Depends<T>` without a manual `Injectable` impl.

A long-standing security gap is closed: all 23 middleware implementations now convert handler errors to responses internally, ensuring post-processing—security headers, CSP, CORS, CSRF, and session cookies—always runs, even on error responses. Framework-level 404 and 405 responses are also routed through the router's middleware chain. Three new built-in authentication middlewares are available: `JwtAuthMiddleware`, `RemoteUserMiddleware`/`PersistentRemoteUserMiddleware`, and `LoginRequiredMiddleware`. The `guard!()` macro enables declarative endpoint-level authorization with composable `&`, `|`, and `!` operators integrated with OpenAPI security generation.

URL resolution gains comprehensive compile-time type safety. `#[url_patterns]` now requires a typed argument (`#[url_patterns(InstalledApp::auth, mode = server|client|unified)]`), validating app labels at compile time against `installed_apps!`. Resolvers are namespaced per application (`urls.server().auth().login()` instead of a flat `urls.auth_login()`), eliminating name collisions. The `#[routes]` macro supports async factory functions with `#[inject]` parameters, enabling DI-constructed routers. Client-side routes are resolvable via `ResolvedUrls::from_global().client()`, and WebSocket handlers gain `#[url_patterns(mode = ws)]` with `urls.ws()` type-safe resolution.

Several developer-experience features also land this release. Hot Module Replacement (`hmr` feature) enables CSS hot-swapping and full-page reloads on Rust changes. `SuspenseBoundary` provides React-inspired lazy loading with configurable fallback components. The `form!` macro gains `SubmitButton` support, `autocomplete` attribute, and automatic CSRF token injection. The MSW-style `MockServiceWorker` module intercepts `window.fetch` at the browser API level for zero-friction WASM testing. The `createsuperuser` command works automatically with any `#[user(full = true)]` model, and the new `RunserverHook` trait enables concurrent service startup (e.g., gRPC) alongside the HTTP server.

Two new infrastructure crates expand the ecosystem. `reinhardt-streaming` delivers Kafka integration across three phases: a low-level producer/consumer API, a declarative macro layer, and per-app typed streaming accessors. `reinhardt-query` gains a Redis command builder with RESP3 output and typestate-validated command construction. On the data-model side, UUID primary keys now use time-ordered v7 generation for database-friendly sorting, and the migration engine gains `SetAutoIncrementValue` and `CreateCompositePrimaryKey` operations. The testkit adds `with_test_di_context()` for isolated, parallel-safe DI contexts in integration tests, and `APIClient` gains an in-process test transport via the `Handler` trait.

### Breaking Changes

- **`#[url_patterns]` requires typed arguments** ([#3670](https://github.com/kent8192/reinhardt-web/issues/3670))
  - Old: `#[url_patterns]`, `#[url_patterns("auth")]`, `#[url_patterns(auth)]`, or `#[url_patterns(client = true, app = "auth")]`
  - New: `#[url_patterns(InstalledApp::auth, mode = server|client|unified)]`
  - Function-name convention: `url_patterns()` → `server_url_patterns()` / `client_url_patterns()` / `unified_url_patterns()`
  - Projects must invoke `installed_apps!` (scaffold now generates `src/config/apps.rs` automatically; `reinhardt startapp` appends new entries there)

### Added

- **`AppLabel::path(&self) -> &'static str`** method on the existing `AppLabel` trait ([#3670](https://github.com/kent8192/reinhardt-web/issues/3670)) — a default implementation returns `Self::LABEL`, so existing unit-struct implementors (`AuthApp`, `ContentTypesApp`, `SessionsApp`) continue to work unchanged. Enum-style implementors (like the `InstalledApp` enum generated by `installed_apps!`) override `path()` to dispatch on `self`.
- **`mode = unified` for `#[url_patterns]`** ([#3670](https://github.com/kent8192/reinhardt-web/issues/3670)) — scans `.server(|s| ...)` and `.client(|c| ...)` closures on a `UnifiedRouter` and emits both `url_resolvers` and `client_url_resolvers` modules.
- **`reinhardt startapp` appends new apps to `src/config/apps.rs`** ([#3670](https://github.com/kent8192/reinhardt-web/issues/3670)) — idempotent on repeat invocations; silently skips when the file is absent (older project structure).
- `ValidatorScope` and `ClientTrigger` enums in `reinhardt-manouche::core::form_node`. Validator rules now carry a `scope` field that is propagated through the typed AST to codegen, where `.validate()` is filtered to server-scoped rules (`Both`, `Server`, `ServerAndClient`). (#3654)

### Fixed

- **`UnifiedRouter::with_namespace` now propagates to the inner `ClientRouter`** ([#3726](https://github.com/kent8192/reinhardt-web/issues/3726)) — previously only the server-side namespace was set, so client-side named routes inside a `UnifiedRouter` were not prefixed with `"<namespace>:"`.

### Deprecated

- `form!` macro: `client_validators: { ... }` is now rejected with a customized compile error that directs users to the unified `validators:` block with `#[client(on = submit|input|blur)]` annotations. The feature had no code generation path in any released version; no runtime behavior is lost. The identifier will be removed from the parser entirely in 0.2.0. See `docs/migration/0.2.0-client-validators-removal.md`. (#3654)

### Migration

1. Add `pub mod apps;` to `src/config.rs` and create `src/config/apps.rs`.
2. Populate `src/config/apps.rs` with `installed_apps! { <label>: "<label>", ... }`.
3. In each app's `urls.rs`:
   - Add `use crate::config::apps::InstalledApp;`
   - Change the attribute to `#[url_patterns(InstalledApp::<label>, mode = server)]` (or `client` / `unified`)
   - Rename `url_patterns` → `server_url_patterns` / `client_url_patterns` / `unified_url_patterns`
4. `reinhardt startapp` now handles step 2 for you on new apps.
