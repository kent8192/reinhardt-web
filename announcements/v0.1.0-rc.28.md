# reinhardt-web v0.1.0-rc.28

## Highlights

This release primarily addresses a high-priority regression in the `reinhardt-urls` crate. After #4242 introduced reactive observation state to `ClientRouter`, the `Rc<RefCell<Vec<Weak<NavigationListener>>>>` and `Rc<Cell<u64>>` fields were left unguarded, making `ClientRouter` (and by extension `UnifiedRouter`) `!Send + !Sync` on native targets. Any consumer that registered `UnifiedRouter` or a wrapping type into a multi-threaded dependency-injection container would fail to compile. These fields are now correctly `#[cfg(wasm)]`-gated, restoring `Send + Sync` on native. A compile-time static assertion (`assert_send_sync::<ClientRouter>()`) is added to `core.rs` to catch future regressions at build time.

Alongside the Send+Sync fix, `on_navigate` and the `__diag_*` diagnostic methods remain callable on native after the cfg-gating, and Copilot review feedback on the same `ClientRouter` surface has been addressed.

The `reinhardt-urls` cross-target builder surface gains a `server_fn` stub on `ServerRouterStub`. Previously, calling `UnifiedRouter::new().server(|s| s.server_fn(marker))` on `wasm32-unknown-unknown` required a `#[cfg(native)]` workaround at every call site; the new no-op stub closes that gap so the builder chain compiles unchanged on both targets.

`reinhardt-conf`'s `SettingsBuilder` gains a new `MergeStrategy` enum (`Shallow` | `Deep`). `build_composed::<T>()` now defaults to `Deep`, meaning environment-specific TOML layers no longer need to redeclare entire sections to override a single key â€” nested siblings are preserved. `build()` retains `Shallow` semantics for backward compatibility, and `MergeStrategy` is re-exported from the `settings::` prelude. An explicit `with_merge_strategy(MergeStrategy::Shallow)` opt-out is available for `build_composed` callers that relied on the previous behaviour.

## Breaking Changes

No breaking changes in this release.
