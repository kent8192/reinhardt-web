# reinhardt-web v0.1.0-rc.15

## Highlights

This release delivers **automatic WASM initialization** for `--with-pages` SPA mode. `StaticFilesMiddleware` now auto-detects wasm-bindgen output pairs (`*.js` + `*_bg.wasm`) in the static directory and injects the initialization script into `index.html` before `</body>`, eliminating manual boilerplate. Projects that prefer the previous behavior can opt out with `.auto_inject_wasm(false)`.

Authentication gains two major improvements. The new `#[user]` macro auto-generates `BaseUser`, `FullUser`, `PermissionsMixin`, and `AuthIdentity` trait implementations from a struct definition, removing approximately 160 lines of boilerplate. `AuthPermission` and `Group` are now proper database models with `ManyToManyField` relationships, and `GroupManager` integrates with `PermissionsMixin` for runtime group permission resolution. Additionally, `JwtAuth` methods now return a typed `JwtError` instead of a generic error, allowing callers to distinguish expired tokens from invalid signatures programmatically; a new `verify_token_allow_expired()` method supports token refresh flows.

The admin panel receives a comprehensive correctness and security overhaul. CSRF token validation is now enforced on all mutation endpoints, model-level permissions are properly respected, and `admin_routes()` auto-registers `AdminSite` in the DI container. Authentication is added to the previously unprotected `get_fields` endpoint, URL path segments are percent-encoded to prevent malformed URLs, bulk delete gains a 1,000-record ID limit against DoS, export responses now indicate when results are truncated, and sidebar active-link highlighting works correctly.

Several reliability improvements land across the ORM and DI layers. `#[field(skip = true)]` skips a field from database column mapping. `field_type_to_metadata_string` adds support for `Vec<T>`, `Value`, and `HashMap` types. The DI framework correctly registers the HTTP request in `request_scope` during `fork_for_request`, and `DatabaseConnection` is now properly propagated into user-provided `InjectionContext` instances. Feature-gate correctness is improved so `reinhardt-core`, `reinhardt-db`, `reinhardt-rest`, and related crates compile with `--no-default-features`. The Rust toolchain is updated to 1.94.1 with MSRV formally set to 1.94.0.

## Breaking Changes

No breaking changes in this release.
