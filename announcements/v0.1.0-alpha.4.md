# reinhardt-web v0.1.0-alpha.4

## Highlights

This release resolves a long-standing WASM build failure (issue #81) in which `tokio` and `mio` — server-side async runtime dependencies — were inadvertently pulled into WASM compilation targets. The fix gates `tokio` and `hyper` behind `#[cfg(not(target_arch = "wasm32"))]` throughout `reinhardt-core` and the signals subsystem (including `batching`, `distributed`, `dlq`, `graphql_integration`, `replay`, and `throttling` modules), and makes the `send_async` method on signals conditional as well. Additionally, `reinhardt-pages` now declares `getrandom` with the `wasm_js` feature and `uuid` with the `js` feature for WASM targets. The result: `cargo check --target wasm32-unknown-unknown --package reinhardt-pages` now succeeds cleanly.

This release also extracts the Manouche DSL — page templates, form definitions, and head element declarations — from `reinhardt-pages-ast` into a new first-class crate, `reinhardt-manouche`. The DSL now has a dedicated home in the crate hierarchy, making it easier to extend independently of the pages module. Users depending on `reinhardt-pages-ast` should migrate by replacing that dependency with `reinhardt-manouche` in `Cargo.toml` and updating imports from `reinhardt_pages_ast::*` to `reinhardt_manouche::*`. The old crate continues to re-export all types with deprecation warnings during the transition period, so existing code keeps compiling without immediate changes.

## Breaking Changes

No breaking changes in this release.
