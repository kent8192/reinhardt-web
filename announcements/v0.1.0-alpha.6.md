# reinhardt-web v0.1.0-alpha.6

## Highlights

This release fixes a critical bug in the `runserver` command where it was unconditionally using `Settings::default()` instead of loading configuration from the project's `settings/` directory. As a result, any customization made through `settings/base.toml` or environment-specific files (e.g., `settings/local.toml`) was silently ignored at startup.

A direct consequence of this bug was that static file serving — introduced in a prior release — never functioned correctly in practice. Because `runserver` always started with empty defaults, `staticfiles_dirs` was always an empty list regardless of what was configured, making it impossible to serve static assets through the development server. Both issues are now resolved: settings load correctly, and static files configured via `staticfiles_dirs` are served as expected.

Settings are now applied in the following priority order (lowest to highest): built-in defaults, `REINHARDT_` environment variables, `settings/base.toml`, and finally `settings/{profile}.toml` (where the profile is controlled by the `REINHARDT_ENV` environment variable, defaulting to `local`).

## Breaking Changes

No breaking changes in this release.
