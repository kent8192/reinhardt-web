# reinhardt-web v0.1.0-rc.26

## Highlights

This release resolves a long-standing SPA navigation regression where `ClientLauncher`'s render Effect failed to re-fire after `Router::push`, causing every route to mount the boot-time view. The fix progressed through diagnosis (#4078, #4102) and culminates in a structural refactor (#4114) that replaces reactive Effect/Signal auto-tracking in the launcher with an explicit `Router::on_navigate` callback list — the same pattern as React Router's `router.subscribe`. This eliminates the failure class that caused regressions #3348, #4075, and #4088. A companion fix (#4110) ensures `on_navigate` observers also fire on browser back/forward (`popstate`), making the subscription primitive safe for use as the launcher's sole render trigger.

`reinhardt-conf` gains opt-in `${VAR}`, `${VAR:-default}`, `${VAR:-}`, and `${VAR:?message}` interpolation for TOML configuration files via `TomlFileSource::with_interpolation(true)`. This allows a single `local.toml` to serve both host-native and dev-container targets without file duplication or verbose environment-variable override prefixes. Separately, `JsonFileSource` and `auto_source` are now `#[deprecated]` and will be removed in 0.2.0; the recommended migration is to `TomlFileSource` or a custom `ConfigSource` implementation.

Several macro and routing correctness fixes ship in this release: the `#[routes]` macro now emits per-app typed client URL accessors (`urls.client().<app>().<route>()`) on `wasm32-unknown-unknown` targets (#4132), `ClientUrlResolver` is correctly brought into scope in generated client accessor code (#4090), and native `UnifiedRouter::mount_unified` now merges child `ClientRouter` routes in addition to `ServerRouter` entries (#4077). The `form!` macro also retains `server_fn` use-statements live on native targets to prevent spurious `unused_imports` warnings under `-D warnings` (#4071).

The `runserver` dev-server command now performs a source-mtime staleness check before skipping a WASM rebuild, preventing stale bundles from being served silently across editing sessions (#4131).
