# reinhardt-web v0.4.0-alpha.1

## Highlights

This is a major pre-release focused on performance and a wave of breaking API refinements across Pages, the ORM, and DI. The HTTP request path is substantially faster: query and path parameters are now lazily parsed and borrowed rather than eagerly materialized, compiled route tables are frozen behind lock-free lookups, and new synchronous handler/router paths avoid boxed futures entirely for common no-middleware routes — cutting per-request allocations dramatically.

Reinhardt Pages gains a large set of reactive improvements: hook dependencies must now be declared explicitly via `deps![...]` or `deps_auto!()`, reactive handles (`Signal`, `Memo`, `Effect`, `Resource`, etc.) are now `Copy` scope-owned keys instead of `Rc`-based handles, and SSR can resolve `use_resource` data asynchronously with progressive Suspense streaming. Routing adds nested layout routes with a persistent `Outlet`, route-level data loaders, and now requires explicit `name = "..."` on route-backed components. Forms get controlled input bindings, DTO-derived `ClientForm` generation, typed action/query helpers, and a versioned structured server-function error contract.

On the data layer, the ORM adds opt-in N+1 query detection, typed JSON model fields, native Rust enum fields with generated CHECK constraints, typed relation traversal for `QuerySet`, and closure-scoped atomic transactions, alongside a new model table-naming convention and unified foreign-key accessors. DI introduces self-keyed provider wrappers, auth adds password-hash policy upgrades (Argon2id/bcrypt), and Pages gains reactive i18n, compile-time accessibility validation, and a component-scoped style DSL. A large portion of the changelog also reflects extensive CI stabilization work to keep this fast-moving pre-release branch releasable.
