# reinhardt-web v0.3.0-rc.6

## Highlights

The migration autodetector now correctly handles field changes that coincide with a model or table rename. Previously, renaming a model while simultaneously adding, altering, or removing fields could produce noisy create/drop operations instead of the intended modify sequence. `RenameTable` is now also ordered before any operations targeting the renamed table, resulting in cleaner, more accurate generated migrations.

A compile-breaking bug in the `#[model]` macro is fixed: generated `{Model}Info` companion structs no longer inherit `#[serde(skip)]` on relation fields (FK, OneToOne, and ManyToMany). The `#[model]` macro injects `#[serde(skip)]` onto relation fields so that only the foreign-key column serializes on the model itself, but that attribute must not propagate to the Info struct's relation payload fields — which caused E0277 compile errors for any model with FK or OneToOne Info fields. Plain-field serde redaction (e.g., `#[serde(skip_serializing)]` on sensitive fields like `password_hash`) is preserved.

Cookie extraction gains first-class support for custom `CookieName` marker types. `CookieNamed<N, String>` and `CookieNamed<N, Option<String>>` now accept any downstream-defined marker, and `CookieStruct<T>` is recognized by both HTTP route and Pages server-function macros. This enables typed extraction for application-specific cookies — such as OAuth callback state — without manually parsing the raw `Cookie` header.

The Pages app scaffold is corrected to place shared `models.rs` at the app root rather than under `server/`, matching what the basis tutorial expects. Fresh scaffolded projects now include the tutorial-required feature defaults and test helpers out of the box, eliminating the need for manual file moves when following the tutorial.

## Breaking Changes

No breaking changes in this release.
