# reinhardt-web v0.1.0-rc.23

## Highlights

This release delivers significant improvements across authentication, database, REST serialization, and the pages runtime.

**Authentication** sees the largest expansion. The new `GenericOidcProvider` lets applications connect any OIDC-compliant identity provider — Keycloak, Authentik, self-hosted GitLab, and others — using only a discovery URL and client credentials, without reimplementing the `OAuthProvider` trait. Alongside it, the OIDC JWK layer gains Elliptic Curve key support (P-256, P-384, P-521), enabling ID token verification for providers that publish ES256 or ES384 keys. A long-standing bug in `GitHubProvider` is also resolved: the GitHub `/user` response was never correctly mapped to `StandardClaims`, meaning GitHub OAuth logins silently returned no user claims on every real-world callback. The GitHub UserInfo endpoint additionally now enforces HTTPS before sending bearer tokens.

**Views and serialization** fix two critical regression-class bugs. `ModelViewSet` and `ReadOnlyModelViewSet` were returning placeholder JSON (`[]`, `{}`) instead of performing real database I/O — making router-backed CRUD effectively non-functional. Separately, `ModelSerializer`'s `MetaConfig` options (`with_exclude`, `with_read_only_fields`, `with_write_only_fields`) and registered sync validators were bypassed entirely at runtime; they now take effect as documented, preventing data exposure such as excluded fields appearing in serialized output.

**Database migrations** fix a silent correctness bug: the migration autodetector tracked added and removed non-PK constraints (`unique_together`, `Check`) but never emitted the corresponding `AddConstraint` / `DropConstraint` operations, allowing model schema and database schema to diverge without warning. This release also introduces opt-in `CustomManager` / `HasCustomManager` traits for Django-style custom object managers, with 53 default-implemented operations and veto hooks for save, delete, and bulk update.

**Pages runtime** extends `ClientLauncher` with three new builder methods: `intercept_links` for built-in SPA link interception, `before_launch`/`after_launch` lifecycle hooks, and `on_path`/`on_path_pattern` for path-driven side effects — eliminating boilerplate that downstream applications previously had to hand-roll.

## Breaking Changes

No breaking changes in this release.
