# reinhardt-web v0.1.0-rc.5

## Highlights

This release delivers a substantial security hardening pass. `StyleTag::to_html()` now escapes closing tag sequences to prevent XSS injection (#1902), CSRF token rotation uses saturating arithmetic to prevent a panic on clock skew (#1903), and composite primary key SQL generation quotes all identifiers to close a SQL injection vector (#1904). Token comparison in the admin layer now uses SHA-256 digest comparison via the `subtle` crate to eliminate timing side-channel length leaks (#1673). `JwtSessionBackend` validates HMAC secret key length per NIST SP 800-107 (#1926), and OAuth2 authorization code exchange now enforces `redirect_uri` matching per RFC 6749 Section 4.1.3 (#1929). The `wasmtime` dependency is downgraded to 36.0.6, resolving four published CVEs (#1924).

Authentication and session correctness improves in several areas. `InMemoryTokenStorage` migrates from `std::sync::RwLock` to `tokio::sync::RwLock` to prevent async-context deadlocks (#1685), non-UUID user IDs now map deterministically via `Uuid::new_v5` instead of generating a fresh random UUID per request (#1671), and the login server function in examples correctly persists session data and regenerates session IDs to prevent session fixation (#1795).

Database and ORM reliability sees significant gains. The `#[model]` macro now correctly propagates `auto_increment`, `nullable`, and `auto_now` parameters to `FieldMetadata`, fixing migration SQL generation (#1741). Schema generation uses `app_label_modelname` table keys to prevent cross-app collisions and no longer silently drops index definitions (#1740). Custom `db_table` names are respected during schema export (#1979), and the insert builder propagates errors instead of panicking on column/value mismatches (#1907).

Two macro utilities are backported from the development branch under the new SP-6 non-breaking addition policy: `pre_validate` enables automatic field validation (#1940), and `ApplyUpdate` provides a derive macro for partial struct updates (#1953). WebSocket and query builder types (`WebSocketConsumer`, `RoomManager`, `Order`, `IntoValue`, and others) are now re-exported from the top-level `reinhardt` crate, eliminating the need for direct sub-crate dependencies (#1923). `UuidConverter` now accepts uppercase UUIDs per RFC 4122 (#1908), and `extract_path_params` correctly handles UUID and slug path segments in addition to numeric IDs (#1977). Over 1,700 new intra-crate integration tests are added across seven crates (#1813–#1817), and the website documentation is corrected to match the actual implementation (#1863).

## Breaking Changes

No breaking changes in this release.
