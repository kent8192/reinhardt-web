# reinhardt-web v0.1.0-alpha.19

## Highlights

This release improves the developer experience for database migrations and finalizes the launch of the official documentation website at [reinhardt-web.dev](https://reinhardt-web.dev).

The database migration subsystem now emits a `tracing::warn!` log when `.sql` files are detected in the migrations directory. Users migrating from Django or other SQL-first frameworks sometimes place `.sql` files there, which were silently ignored. In addition, the "No migrations to apply" message now includes guidance on the required `.rs` file format, directory structure, and the `makemigrations` command, making the expected workflow immediately clear without consulting external documentation.

The official documentation website at `reinhardt-web.dev` is now fully operational. Two infrastructure fixes were required to get there: the Cloudflare Pages project's production branch was not set to `main`, causing all deployments to be classified as preview builds, and the necessary DNS records for the custom domain were missing from the Terraform configuration. Both issues are now resolved, and `reinhardt-web.dev` serves the canonical production build on every push to `main`.

All README documentation links—both in the English README and in seven localized translations (Japanese, Simplified Chinese, Traditional Chinese, Russian, Ukrainian, Persian, and Arabic)—now point to the official website URLs on `reinhardt-web.dev`. The previously repository-hosted documentation files that had been fully migrated to the static site have been removed to eliminate split maintenance and the risk of content divergence.

## Breaking Changes

No breaking changes in this release.
