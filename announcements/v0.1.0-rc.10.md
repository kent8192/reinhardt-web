# reinhardt-web v0.1.0-rc.10

## Highlights

The `reinhardt::graphql` module now re-exports commonly used `async_graphql` types — including `Object`, `SimpleObject`, `Schema`, `Context`, and `Error` — through the reinhardt facade. Users can rely on `reinhardt::graphql` for most imports instead of depending directly on `async-graphql`, reducing coupling to the underlying GraphQL library and easing future migrations.

The `#[derive(Validate)]` macro adds `#[validate(range(min = N, max = M))]` support for numeric fields. Developers can now declare min-only, max-only, or combined min/max bounds directly on struct fields without writing manual validation logic.

Two macro compile errors are resolved. The `#[routes]` macro previously emitted `#[cfg(feature = "client-router")]` guards that were evaluated in the downstream crate's feature context rather than the library's, causing build failures in projects that do not enable `client-router`. Additionally, using `pre_validate = true` alongside destructuring extractors such as `Json(body): Json<T>` no longer produces a "use of moved value" compile error — the generated code now extracts into a temporary, validates, then destructures.

Feature-flag reliability in the commands crate sees several fixes. The `openapi-router` feature is now correctly propagated from the root crate to `reinhardt-commands`, so `/api/docs` responds as expected when the feature is enabled; the runserver startup banner is likewise gated on the feature to avoid misleading output. `MigrateCommand` and `connect_database()` now respect `postgres`/`sqlite` feature guards, matching the existing `IntrospectCommand` pattern and preventing compile errors when only one backend is active.

Project template quality is also improved: `installed_apps! {}` with empty input compiles correctly under Rust 2024 edition, restful template settings files are renamed to the `.example.toml` convention so git tracks them properly, and dead-code template files containing invalid `installed_apps!` entries are removed.

## Breaking Changes

No breaking changes in this release.
