# reinhardt-web v0.3.0-rc.4

## Highlights

This release tightens the Pages scaffolding experience with two targeted fixes uncovered during basis tutorial validation. First, the Pages project generator now correctly injects the required `minimal` facade runtime when a user supplies an explicit custom feature list, while still respecting any explicitly chosen database backend. Previously, projects bootstrapped with `--default-features false` and a hand-picked feature set could omit the facade dependency needed by generated code, causing compilation failures that were difficult to diagnose.

Second, generated Pages projects switch their WASM build pipeline from hand-picked `wasm-bindgen` input files to `wasm-pack` library builds. The generated `Makefile.toml` now invokes `wasm-pack` and copies the resulting browser bundle into `dist/` after `collectstatic`, eliminating a class of wrong-`.wasm`-file errors that surfaced when a native `manage` binary was present alongside the WASM target.

The Pages app scaffold layout is also reorganized to match the current component-macro routing model. App root files now act as module declarations, with implementation files living under app-local directories. The obsolete `pages.rs` wrapper and the broad project-level `shared` module are removed; in their place, generated apps gain explicit `serializers/`, `server_fn/`, `services/client/`, `services/server/`, and server-side `forms/` scaffolding directories with `.gitkeep` preservation. This makes client/server ownership immediately clear in freshly generated projects.

Finally, the basis tutorial and quickstart documentation are updated to reflect the current 0.3.0 scaffolding structure, covering model-generated DTOs, `#[injectable]`, `Depends`, `CurrentUser`, SQL seeding, and `installed_apps!` usage.

## Breaking Changes

No breaking changes in this release.
