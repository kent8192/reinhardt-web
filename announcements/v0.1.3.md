# reinhardt-web v0.1.3

## Highlights

This release adds two convenience type aliases to the dependency injection system: `DependsResult<T, E>` (equivalent to `Depends<Result<T, E>>`) and `DependsOption<T>` (equivalent to `Depends<Option<T>>`). These eliminate the verbosity of spelling out the full wrapper form when injecting fallible or optional dependencies, and are re-exported from the `reinhardt` facade alongside `Depends`. The underlying `#[injectable_factory]`, `#[injectable]`, and route/server-function macros are extended to resolve these aliases correctly, so they work end-to-end without any additional boilerplate.

On the pages side, `ServerFnError` gains a new `message(&self) -> &str` accessor that returns the human-readable error text without the variant prefix that `Display` adds (e.g., `"Application error: …"`). A companion free function, `parse_server_error_message(raw: &str) -> String`, deserializes JSON-serialized `ServerFnError` envelopes and extracts the inner message, falling back to the raw string when parsing fails. Both are available as `reinhardt::pages::parse_server_error_message`, removing the need for hand-rolled parsing helpers at call sites.

Several correctness issues in the `reinhardt-admin fmt` command are resolved in this release. The formatter now runs rustfmt on files that contain no DSL macros and on files marked with `reinhardt-fmt:ignore-all`, both of which were previously skipped entirely. `find_rustfmt_config` no longer stops at the first `Cargo.toml` it encounters, so workspace `rustfmt.toml` settings (`max_width`, `hard_tabs`, etc.) are applied consistently by `fmt` and `fmt-all` alike. The `fmt-all` command additionally skips nested workspace roots, resolving a long-standing oscillation where `fmt-all` and `fmt examples` demanded opposite formatting for the same file.

The DSL formatting backend (tree-sitter + Topiary for `page!`, `form!`, and `head!` macro bodies) is now shipped as a dedicated `reinhardt-formatter` crate, separating it from the publishable `reinhardt-admin-cli` package and resolving the `wasm-bindgen` version conflict that previously blocked crates.io publication.
