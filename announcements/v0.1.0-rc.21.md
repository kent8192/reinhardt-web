# reinhardt-web v0.1.0-rc.21

## Highlights

This release fixes a compilation error introduced when scaffolding new apps with pages. Running `reinhardt-admin startapp --with-pages` generated an `app-root` `lib.rs` that contained a stray top-level `pub mod ws_urls;` declaration. Because `ws_urls` lives at `src/apps/<name>/urls/ws_urls.rs` and is already declared inside `urls.rs`, the duplicate top-level entry caused the compiler to look for a non-existent `src/apps/<name>/ws_urls.rs`, producing an immediate `E0583` error. The fix removes the erroneous declaration from both the standard and workspace app-pages templates.

This release also completes the three-part version-sync automation series. All 28 crate-level `README` files now carry `<!-- reinhardt-version-sync -->` markers covering 51 installation code blocks, and the version-update script has been switched from a static file list to a `find`-based enumeration — meaning any new crate added to the workspace is picked up automatically with no manual maintenance. The release pipeline now runs `update-version-refs.sh` as part of the Release PR itself, so documentation version references are current the moment the Release PR is merged rather than lagging by one cycle.

## Breaking Changes

No breaking changes in this release.
