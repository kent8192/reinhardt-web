# reinhardt-web v0.1.0-alpha.18

## Highlights

This release introduces the official reinhardt-web website at **reinhardt-web.dev**, built with Zola — a Rust-native static site generator. The site ships with a full documentation structure covering quickstart guides, tutorials, API reference, and a cookbook, all accessible through a unified weight-based sidebar navigation. The "Why Reinhardt?" section now showcases 12 feature cards, including highlights for the WASM plugin system, Django-style CLI, FastAPI-style dependency injection, and auto-generated OpenAPI with a browsable API explorer.

The website also receives a polished UI pass: syntax highlighting migrates to highlight.js for richer theme support, the color palette expands with new web font variables, and standalone pages (changelog, security) are properly centered. Accessibility improvements, security headers, SRI integrity attributes, and FOUC prevention are added to the asset pipeline. Documentation content across the site is audited and corrected — fabricated API examples, incorrect import paths, stale repository URLs (`reinhardt-rs` → `reinhardt-web`), and broken docs.rs links are all fixed.

reinhardt-web adopts the **BSD 3-Clause License**, aligning with Django's license and making the project's philosophical relationship to Django clearer to the community. The dual MIT/Apache-2.0 license files are replaced with a single BSD 3-Clause `LICENSE` file, updated consistently across all 51 affected files including every `Cargo.toml`.

On the infrastructure side, workspace entries for two never-existing placeholder crates (`reinhardt-postgres`, `reinhardt-events`) and the unpublished `reinhardt-settings-cli` crate are removed, and stale `reinhardt-settings` references are corrected to `reinhardt-conf` throughout docs and source comments.

## Breaking Changes

No breaking changes in this release.
