# reinhardt-web v0.1.0-alpha.13

## Highlights

This release introduces a comprehensive OAuth2/OIDC social authentication framework via the new `reinhardt-auth` crate. The implementation covers the full authorization code flow — from generating authorization URLs through token exchange and refresh — with built-in support for Google, GitHub, Apple, and Microsoft as first-class providers.

Security is treated as a first-class concern throughout the design. PKCE (RFC 7636) with SHA256 challenge generation protects against authorization code interception attacks, state-based CSRF tokens (with a 10-minute TTL) guard the redirect flow, and nonce validation for OIDC flows prevents replay attacks. RSA signature verification for ID tokens is handled via `jsonwebtoken`, keeping the trust chain intact end-to-end.

The architecture is intentionally modular: core abstractions, protocol flows, and provider integrations live in separate layers, making it straightforward to add custom providers or swap out individual components. OIDC discovery is fully automated through `.well-known/openid-configuration`, and JWKS keys are cached with a 1-hour TTL and automatic rotation so applications stay up to date without manual intervention.
