# reinhardt-web v0.1.0-alpha.9

## Highlights

This release adds first-class support for the `reinhardt-dentdelion` plugin system through the `reinhardt` facade crate. Users can now enable the plugin API with a single optional feature flag — `features = ["dentdelion"]` — rather than taking a direct dependency on `reinhardt-dentdelion`. The `dentdelion` feature is also included in the `full` feature preset, so projects opting into the complete feature set get plugin support automatically.

This change is particularly useful for plugin authors in the ecosystem: a delion plugin can now declare one unified dependency on `reinhardt` with `default-features = false, features = ["dentdelion"]`, keeping dependency graphs clean and avoiding version mismatches between the facade and the plugin subsystem crates.

Additionally, a doctest import path in the `dentdelion` module is corrected to reference the `prelude` module properly, ensuring documentation examples compile and run without errors.

## Breaking Changes

No breaking changes in this release.
