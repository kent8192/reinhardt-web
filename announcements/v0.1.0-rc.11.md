# reinhardt-web v0.1.0-rc.11

## Highlights

This release focuses on correctness in the database migration engine and Cargo feature graph, addressing several bugs surfaced after the `full` → `standard` default-feature change.

The migration engine receives two important bug fixes. Plain string field defaults such as `default = "pending"` were previously emitted as bare SQL identifiers (`DEFAULT pending`), which PostgreSQL interprets as a column reference and rejects. The migration engine now auto-quotes these as SQL string literals (`DEFAULT 'pending'`) while correctly leaving SQL constants (`CURRENT_TIMESTAMP`) and function calls (`NOW()`, `uuid_generate_v4()`) unquoted. Separately, the `makemigrations` code generator fixes three classes of compile errors in generated migration files: `&str`/`String` type mismatches in `quote!` output, missing struct fields (`state_only`, `database_only`, `swappable_dependencies`, `optional_dependencies`), and data loss where `old_definition` and `mysql_options` were silently discarded during `AlterColumn`/`AddColumn` round-trip codegen.

The default Cargo feature is changed from `full` to `standard`. The `standard` feature provides database, REST, auth, middleware, and pages without pulling in every optional crate (admin, GraphQL, WebSockets, i18n, mail). Users who need everything can still opt in with `features = ["full"]`.

A cascade of feature-propagation gaps exposed by the default-feature change are resolved. The `db-postgres`, `db-cockroachdb`, and `db-mysql` features now correctly propagate their backend flags to `reinhardt-commands`. MySQL URL scheme handling (`mysql://`) is also added to the `migrate` command's connection dispatch logic, fixing a type-inference error (`E0282`) that occurred when only the `mysql` feature was enabled.

Documentation for the `makemigrations` command is expanded to cover the `--force-empty-state` flag and the `linkme` crate dependency required by the `collect_migrations!` macro.

## Breaking Changes

No breaking changes in this release.
