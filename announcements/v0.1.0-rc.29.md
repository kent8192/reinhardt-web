# reinhardt-web v0.1.0-rc.29

## Highlights

This release ships first-class DI testing support. `DependencyRegistry::register_override` (feature `testing`) pairs with a new `OverrideGuard` that automatically restores the original factory on drop. A new `reinhardt-testkit-macros` crate provides the `with_di_overrides!` proc-macro, re-exported through `reinhardt-testkit` via `DiOverrideBuilder`, making it straightforward to swap singletons and transient services in `#[rstest]` tests without unsafe registry manipulation.

Several subsystems that previously bypassed the settings system now flow through it. REST versioning gains a `VersioningSettings` fragment under `[rest_versioning]`, retiring the `REINHARDT_VERSIONING_*` env vars. `BrokenLinkEmailsMiddleware::from_settings` removes a per-request env read, and a new `HasCommonSettings` trait with `DatabaseConnection::database_url_from` lets commands resolve the database URL from a loaded `ProjectSettings` handle. Project scaffolds now default to `build_composed` (Deep merge), so environment overrides no longer require redeclaring an entire table to change a single key.

The `reinhardt-test/msw` module is repaired for web-sys 0.3.98 and js-sys API changes, and the `reinhardt-web` facade gains a top-level `msw` feature flag. Two bugs in `#[server_fn]`'s WASM marker-module emission are fixed, unblocking typed MSW handler tests end-to-end. A new `reinhardt-router` crate extracts the `VersionedRouter` trait to break the longstanding `reinhardt-urls` ↔ `reinhardt-rest` circular dependency, replacing the dead-code versioning stubs with real generic implementations.

The SPA link interceptor now correctly walks up the DOM to find an enclosing `<a>` when a click targets an inner text or child element, and exposes `Router::push` failures via `console.warn`. `ModelForm::is_valid` previously returned `true` unconditionally; it now runs the full field-validation pipeline. Non-integer primary keys (e.g., `Uuid`) no longer receive `auto_increment` in generated migrations, fixing a SQLite `AUTOINCREMENT is only allowed on an INTEGER PRIMARY KEY` crash. The `startapp --with-pages` scaffold now generates the canonical `urls/server_urls.rs` + `urls/client_router.rs` submodule layout, closing a gap present since rc.19.

## Breaking Changes

No breaking changes in this release.
