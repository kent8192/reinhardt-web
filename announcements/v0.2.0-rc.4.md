# reinhardt-web v0.2.0-rc.4

## Highlights

The admin interface receives a significant bug-fix pass: the tutorial admin panel now supports full browser CRUD, with authentication properly connected to the example user model, Add and Delete buttons wired to server functions, and numeric record IDs preserved in JSON rendering so action links and form values work correctly.

`StaticFilesMiddleware` now injects the WASM auto-loader when serving a directory `index.html` directly, regardless of whether SPA fallback mode is enabled. Previously, the loader was only injected through the SPA fallback path, causing apps that serve `/` as a concrete directory index with `spa_mode` disabled to receive a raw loading shell without the WASM bootstrap.

Two configuration improvements land in this release. The settings system gains typed schema references for embedded (nested) configuration nodes, with automatic wrapper handling for `Option`, `Vec`, `HashMap`, `BTreeMap`, `IndexMap`, and `Box`. Recursive required-field validation now reports the full generated path for missing nested leaves. In addition, `SecretString` TOML fields now accept `{ env = "VAR_NAME" }` and `{ file = "/path/to/secret" }` source maps alongside the existing plain-string and `{ secret = "..." }` formats; file-backed secrets automatically trim trailing newlines for Docker/Kubernetes compatibility.

The `reinhardt-web` facade crate adds `auth-social` and `commands-server` feature flags, allowing downstream applications to access social authentication and runserver hook surfaces without pulling in implementation crates directly. Finally, `CurrentUser<U>` is established as the canonical authenticated-user extractor; `AuthUser<U>` continues to work through the 0.2 cycle as a deprecated compatibility wrapper scheduled for removal in 0.3.

## Breaking Changes

No breaking changes in this release.
