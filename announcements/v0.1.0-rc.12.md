# reinhardt-web v0.1.0-rc.12

## Highlights

This release introduces **ergonomic auth extractors** as the recommended authentication pattern. `AuthInfo` provides a lightweight extractor for reading `AuthState` without a database query, while `AuthUser<U>` replaces `CurrentUser<U>` with cleaner tuple-struct destructuring and no boilerplate `use_inject = true` annotation. The DI layer now auto-detects `#[inject]` on route handlers and adds an `Option<T>` blanket `Injectable` implementation for optional dependencies. A new `validate_auth_extractors()` function surfaces DI misconfiguration at startup rather than silently falling back to anonymous users. `CurrentUser<U>` is deprecated but continues to work.

Several **security vulnerabilities are patched** in this release. Two HIGH-severity SQL injection vectors in `reinhardt-db` are fixed: unquoted identifiers in reverse migration SQL and unescaped single quotes in `InsertBuilder`/`UpdateBuilder`. The `RequestService` error handler now routes through `SafeErrorResponse`, preventing internal details such as database connection strings and file paths from leaking in HTTP response bodies. CORS middleware now correctly emits `Vary: Origin` when wildcard origins are combined with `allow_credentials: true`, preventing cache poisoning.

The **migration tooling** gains two notable additions. `makemigrations --merge` detects divergent migration graphs (multiple leaf nodes per app) and generates an empty merge migration, mirroring Django's workflow. The testkit adds `postgres_with_migrations_from_dir()` using `FilesystemSource`; the older `global_registry`-based fixtures are deprecated alongside the `collect_migrations!` macro.

Additional fixes address `AuthState::from_extensions()` failing to find a directly-inserted `AuthState`, `ElementBuilder::build()` dropping event handles immediately via RAII, `ChunkedUploadManager` accepting duplicate chunk numbers, `FilesystemSource` silently returning empty migrations for non-existent directories, and the `cli-tools` feature preset failing to compile without `core`/`di` features. The new **`manage introspect`** command outputs structured YAML/JSON application metadata for PaaS platform resource inference.

## Breaking Changes

No breaking changes in this release.
