# reinhardt-web v0.1.0-alpha.5

## Highlights

This release resolves a series of interconnected circular dev-dependency issues that were blocking `cargo publish` and preventing the framework from being distributed on crates.io. Users who install reinhardt-web from crates.io will now receive a correctly packaged set of crates without encountering resolution errors.

The core problem stemmed from several workspace crates carrying dev-dependencies that formed publish-time cycles — for example, `reinhardt-http` and `reinhardt-urls` each referencing the other, and `reinhardt-openapi-macros` indirectly cycling back through `reinhardt-rest`. These cycles are now broken by removing genuinely unused dev-dependencies and, where tests still need shared types, re-exporting them through `reinhardt-test` instead. `ServerRouter` is now re-exported from `reinhardt-test`, keeping test code functional without reintroducing the cycle.

Additionally, the CI pipeline now correctly skips the publish dry-run check on `release-plz-*` branches. Because release-plz bumps all interdependent crates simultaneously before any of the new versions exist on crates.io, the dry-run check was producing false failures on release PRs. The check continues to run on all ordinary development PRs, maintaining its value as an early warning for newly introduced circular dependencies.

Finally, the `reinhardt-test` workspace dependency now carries an explicit version, satisfying Cargo's requirement that all non-dev dependencies specify a version when publishing — a requirement that had silently blocked the `reinhardt-web` top-level crate from being packaged.

## Breaking Changes

No breaking changes in this release.
