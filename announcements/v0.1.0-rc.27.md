# reinhardt-web v0.1.0-rc.27

## Highlights

**WASM cross-target support is now complete.** Downstream wasm SPA consumers can use `#[routes]`, `#[url_patterns]`, `#[app_config]`, and `installed_apps!` without `#[cfg(native)]` workarounds. The `ResolvedUrls` typed accessor cascade is fully reachable from wasm32 call sites, and `ServerRouterStub` now carries a complete no-op builder surface so `mode = unified` closures compile uniformly on wasm targets. `ResolvedUrls` is also now available via an explicit `pub use` from the macro expansion, removing the need for consumers to reach into underscore-prefixed modules.

**A recurring SPA navigation regression class is resolved at the root cause.** `Router::push` and `Router::replace` now serialize `history.state` as a structured JS object rather than a JSON string, so `history.state.route_name` is no longer `undefined` to external callers. The `client_router::history` module is deduplicated to a single source of truth in `reinhardt-urls`, eliminating the mirror-and-drift pattern that allowed the regression to recur across multiple releases.

**The development hot-reload loop is faster and more reliable.** `runserver --with-pages` now rebuilds the WASM bundle by default on every startup and hot-reload restart — use `--no-override-wasm` to preserve the skip-if-present behavior. `Cargo.lock` changes now trigger a rebuild, WASM and server pipelines run in parallel, and failed builds no longer terminate the watcher.

**TOML settings interpolation defaults have changed (breaking).** `TomlFileSource` now enables `${VAR:-default}` expansion by default. A new typed coercion layer converts interpolated strings to their target Rust types at deserialization time, so `port = "${PORT:-5432}"` deserializes directly to `u16` without manual parsing.

**Several high-impact bugs are fixed.** SQLite migrations now emit `INTEGER PRIMARY KEY AUTOINCREMENT` regardless of field width, unblocking `BigAutoField`-based projects. `manage createsuperuser --noinput` now reads a password from `REINHARDT_SUPERUSER_PASSWORD`, enabling automated provisioning. Superuser initialization no longer writes the nil UUID as the primary key. A false-positive `[reinhardt-conf] Warning: flat key` message that appeared on every `manage` invocation is suppressed. The autoreload parent no longer runs `on_server_start` hooks, preventing `EADDRINUSE` on the child's HTTP bind.
