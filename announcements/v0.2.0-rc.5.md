# reinhardt-web v0.2.0-rc.5

## Highlights

The ORM gains a significant ergonomics upgrade in this release. `FieldRef` now exposes Django-style lookup helpers — `.exact`, `.iexact`, `.icontains`, `.starts_with`, `.is_in`, `.range`, `.year().gte`, regex/iregex, and PostgreSQL-specific array, JSONB, and range predicates — bringing type-safe, composable query expressions to the query builder. Composite filter combinators allow multiple predicates to be combined in a single `.filter()` call. Additionally, `#[model]` builders now support opt-in setter overrides for macro-managed fields such as primary keys and timestamps, making data-import and bridge flows possible without abandoning the ergonomic builder API.

Project scaffolding receives two notable additions. `reinhardt-admin startproject` now offers interactive Reinhardt version and feature-flag selection instead of hard-coding `features = ["full"]`, and a new `reinhardt-admin configure` command allows existing projects to update their `Cargo.toml` facade dependency settings after the fact. Alongside this, a new `manage infra` command family provisions and manages local PostgreSQL and Redis containers via the Docker Engine API (bollard), persisting resolved endpoints in `.reinhardt/local-infra.json` — which is now automatically excluded from generated project `.gitignore` templates.

Hot-reload and build feedback loops are substantially faster. The watcher now performs target-specific rebuild selection: Pages-only edits skip the native server rebuild, and server-only edits skip the WASM compilation step. Static `page!(|| { ... })` blocks support hot patching without a full rebuild, and browsers receive an HMR reload notification only after a successful rebuild. A new build-loop benchmark harness tracks cold workspace, Pages WASM, server, and hot-reload target-selection scenarios.

Several correctness fixes round out the release: `reinhardt::auth` is now accessible as a short umbrella facade alias instead of the verbose `reinhardt::reinhardt_auth` path; `runserver --with-pages --wasm-optional` correctly forwards WASM startup flags to the autoreload child process; LIKE predicates now emit an explicit `ESCAPE '\\'` clause; and a time-based permission test no longer fails intermittently at minute boundaries.

## Breaking Changes

No breaking changes in this release.
