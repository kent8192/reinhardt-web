# reinhardt-web v0.2.0-rc.2

## Highlights

The `page!` macro receives a comprehensive Manouche DSL v2 overhaul. Expressions and control-flow are now unconditionally wrapped in `Page::reactive(move || ...)`, eliminating explicit `watch { }` blocks and fixing helper-routed Signal reads. Reactive hooks (`use_effect`, `use_memo`, `use_callback`, etc.) require an explicit dependency array aligned with React.js semantics, and the compiler now verifies completeness at compile time via a new `hook_deps_validator` pass. Bare-identifier shorthands in element bodies are a hard error; use `{name}` instead. The DSL gains React-style `Component { prop, @event: handler, child { ... } }` brace invocation with named `$slotname { ... }` slots and `@key(...)` for stable list rendering. New runtime primitives include `ErrorBoundary`, `RouterOutlet` with route metadata, action batching with optimistic rollback, and a unified `use_resource(fetcher, deps)` hook that fixes a latent dependency-refetch bug in the previous API.

The ORM query API consolidates: `Manager::filter`, `QuerySet::filter`, and `CustomManager::filter` now accept a single `impl Into<Filter>`, replacing the legacy three-argument form. Custom managers unify under `Model::objects()` via `#[model(manager = MyManager)]`, deprecating the separate `HasCustomManager` trait. `Model::new()` becomes a zero-argument alias for the typestate builder `Model::build()`. The `#[model]` macro auto-generates a `{Model}Info` companion DTO with bidirectional `From` conversions. `Operation::to_reverse_sql` now returns `Vec<String>`, enabling multi-statement DDL rollback and restoring NOT NULL fidelity on CockroachDB. The `manage migrate` command gains Django-style migrate-with-target semantics, `--plan`, and `--fake` flags.

Configuration across nine crates adopts the `#[settings(fragment = true)]` pattern as the canonical API, deprecating ad-hoc `XxxConfig` structs in `reinhardt-tasks`, `reinhardt-server`, `reinhardt-grpc`, `reinhardt-deeplink`, `reinhardt-websockets`, `reinhardt-auth`, `reinhardt-middleware`, and `reinhardt-mail`. Legacy `reinhardt-conf` advanced settings types (`AdvancedSettings`, `Settings`, `JsonFileSource`) are removed entirely. `LocalStorage` now rejects path traversal attempts with `StorageError::InvalidPath`, and `StorageError` gains `#[non_exhaustive]`. Google Cloud Storage and Azure Blob Storage backends ship as fully implemented feature-gated additions.

Sixty APIs deprecated during the 0.1.x RC cycle are removed across twelve crates, covering old URL resolver codegen, `SeaRc<T>`, `Injected<T>`/`OptionalInjected<T>`, `SessionStoreRef`, `OpenApiConfig`, panicking URL reverse helpers, `MockFetch`, `TestUser`, and more. The DI layer gains `Injectable` impls for `Path<T>`, `Query<T>`, and `Json<T>` extractors; enforces scope-hierarchy at resolution time; and adds `DependsResult<T, E>` / `DependsOption<T>` type aliases. Client routing simplifies: `route_pathN` collapses into arity-generic `route_path`, and the type-safe `#[url_patterns]` / `ResolvedUrls` / `ClientUrlReverser` layer is replaced by a unified string-based `UrlReverser` auto-populated at `register_router()` time.
