# reinhardt-web v0.1.0-rc.3

## Highlights

This release focuses on fixing critical compilation errors introduced in the project and app scaffolding templates, ensuring that newly generated projects work out of the box.

Running `reinhardt-admin startproject` and `reinhardt-admin startapp` previously produced code with 11 distinct compilation errors, making it impossible for new users to follow the Quick Start guide without manually patching generated files. This release corrects all of these issues: the `url_patterns` re-export is updated to the correct `routes` name, `Profile::from_str()` is replaced with `Profile::parse()`, `UnifiedRouter::new()` is used in place of the removed builder pattern, missing `serde_json` and `tokio` dependencies are added to the generated `Cargo.toml`, and self-referencing module patterns are removed from generated `models.rs`, `views.rs`, `admin.rs`, and `serializers.rs`. The `app_config` derive macro interaction is also corrected so that `#[app_config(...)]` no longer conflicts with a redundant `#[derive(AppConfig)]`. A new `crate_name` template variable (hyphen-to-underscore conversion) is also introduced to handle hyphenated project names properly.

Additionally, an internal publishing pipeline issue is resolved: starting with Cargo 1.84, `cargo publish` resolves dev-dependencies during packaging even when `--no-verify` is passed, causing `release-plz` to publish crates in the wrong order. The fix moves `reinhardt-test` to an optional dependency for non-cyclic crates and uses path-only dev-dependencies (without a version field) for crates that would otherwise create circular references, ensuring correct publish ordering going forward.

## Breaking Changes

No breaking changes in this release.
