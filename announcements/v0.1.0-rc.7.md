# reinhardt-web v0.1.0-rc.7

## Highlights

This release focuses on improving WASM target support and fixing several runtime errors that affected project bootstrapping and example compilation.

`UnifiedRouter` is now available on `wasm32-unknown-unknown` targets. Previously, the unified routing abstraction was gated behind a non-WASM `cfg`, making WASM builds unable to register client-side routes through the same API. A new `ServerRouterStub` variant enables `UnifiedRouter` to work transparently on both server and client targets without changes to application code. The `UnifiedRouter` re-export in the prelude is also now correctly gated behind the `client-router` feature flag, preventing compile errors in crates that do not enable it.

The `form!` macro receives a critical fix: field value `let` bindings were previously wrapped in a nested block scope, making them inaccessible inside the `spawn_task` async closure used for form submission handlers. This caused "cannot find value" compile errors in any form that referenced field values in its submission logic.

Running `runserver` or `collectstatic` with a minimal project configuration no longer panics. The commands previously expected `middleware`, `root_urlconf`, and `media_root` to be present in settings but provided no defaults, resulting in crashes. These defaults are now supplied automatically.

Projects using a lib/bin split (`src/lib.rs` + `src/bin/manage.rs`) could silently lose all URL patterns at runtime with a "No URL patterns registered" error. The linker discards `inventory::submit!` registrations generated by `#[routes]` when the binary does not reference the library crate. The affected examples now use `use crate as _` to force-link the library; users with the same project structure should apply the same pattern to their `manage.rs`.

## Breaking Changes

No breaking changes in this release.
