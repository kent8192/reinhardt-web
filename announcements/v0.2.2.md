# reinhardt-web v0.2.2

## Highlights

This release brings two significant developer-experience improvements to the toolchain. The dependency injection system gains `DependsResult<T, E>` and `DependsOption<T>` sugar aliases that reduce verbosity when injecting fallible factory outputs — route macros and server functions both resolve these aliases end-to-end. The `reinhardt-admin fmt` command receives a set of correctness fixes: DSL-skipped and `// reinhardt-fmt:ignore-all` files now receive rustfmt when `--with-rustfmt` is enabled, `rustfmt.toml` lookup now traverses past sub-crate `Cargo.toml` files to reach the workspace root, and `fmt-all` now skips nested workspaces, eliminating a long-standing oscillating format conflict between `fmt` and `fmt-all`.

Pages hot-reload latency drops substantially. Static `page!(|| { ... })` edits in WASM-owned source paths now apply via a compile-free HMR hot-patch path (measured at ~99% faster than a full WASM rebuild for supported template edits). Server-only edits skip the WASM pipeline; client-only edits skip native server rebuilds. Browser reload is dispatched only after the selected rebuild succeeds. Cold workspace builds are ~17% faster overall after dependency pruning in the Pages/WASM graph.

Several correctness fixes land for the ORM and HTTP layers. The migration autodetector now emits `RenameColumn` for unambiguous field renames instead of destructive `AddColumn + DropColumn` pairs, and halts explicitly on ambiguous candidates. `ModelViewSet` and `ReadOnlyModelViewSet` now wire the declared serializer type `S` by default rather than silently falling back to `ModelSerializer`, preventing raw field exposure. The `#[model]` macro preserves `#[serde(...)]` field attributes on generated `{Model}Info` structs, and `#[user]` now compiles inertly on `wasm32-unknown-unknown` for shared model types.

Two security fixes are included: vendor asset SHA-256 integrity is now verified before any file is written to disk, and Azure SAS tokens configured for backend operations are no longer returned by temporary URL generation. The in-memory session store cleanup is corrected to use an amortized schedule, preventing unbounded memory growth under sustained cookieless requests.

Pages project scaffolding now defaults to SQLite and a minimal feature set, so `startproject --with-pages` reaches a running tutorial checkpoint without requiring database containers.
