# reinhardt-web v0.1.0-alpha.2

## Highlights

This release introduces automatic static file serving in the development server. When `debug = true`, `runserver` now serves files from all configured `staticfiles_dirs` under the `static_url` prefix, with correct MIME type detection for JavaScript, CSS, HTML, JSON, images, fonts, and WebAssembly. Path traversal attacks are blocked, and conflicts between files that appear in multiple source directories are detected and reported with clear error messages.

A new `reinhardt-openapi` crate extracts the `OpenApiRouter` wrapper that previously caused a circular dependency between `reinhardt-urls` and `reinhardt-rest`. Users can opt in via the `openapi-router` feature flag on `reinhardt` and `reinhardt-commands`. Once enabled, documentation endpoints are served at `/api/docs`, `/api/redoc`, and `/api/openapi.json`.

The static files module in `reinhardt-utils` is renamed from `r#static` to `staticfiles`, eliminating the raw-identifier syntax that Rust requires for reserved keywords. The feature flag is correspondingly renamed from `static` to `staticfiles` in both `reinhardt-utils` and `reinhardt-test`. This is a breaking change — see the migration guide in PR #115 for the required import path updates.

Several sub-crates (`reinhardt-utils`, `reinhardt-conf`, `reinhardt-pages`, `reinhardt-test`) advance to v0.1.0-alpha.4 following publish-workflow corrections. The README badges now correctly link to the `reinhardt-web` crate on crates.io and docs.rs.

## Breaking Changes

No breaking changes in this release.
