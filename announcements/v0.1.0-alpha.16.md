# reinhardt-web v0.1.0-alpha.16

## Highlights

This release delivers a sweeping security hardening pass across the reinhardt-web framework. Admin endpoints now enforce authentication and authorization uniformly — previously, routes were accessible without credential verification. SQL injection vulnerabilities in `SimpleSearchBackend` and savepoint name construction are fixed through parameterized queries. Batch injection fixes close path traversal, SSRF, and email header injection vectors in the admin layer. The `ModelAdmin` default permission model switches to deny-by-default, requiring explicit grants rather than implicit allow.

The authentication subsystem (`reinhardt-auth`) receives substantial hardening: passwords migrate to Argon2id hashing, all comparisons use constant-time functions to eliminate timing side-channels, and TOTP upgrades to SHA-256. Session cookies are issued with `Secure`, `HttpOnly`, and `SameSite=Strict` attributes. The new `SecurityMiddleware` bundles configurable allowed-hosts enforcement, CSRF protection, and session hardening into a single composable unit.

Memory safety improves throughout: `mem::zeroed`, `Box::leak`, and `mem::forget` patterns are replaced with safe alternatives (`Option<T>`, `Arc`, `ManuallyDrop`). The DI cycle-detection mechanism migrates from `thread_local!` to task-local storage, preventing deadlocks when resolution spans `.await` points. CSP nonces are now generated via `OsRng` rather than a predictable source. GraphQL query depth and complexity limits, WebSocket rate limiting, and connection timeouts guard against resource exhaustion.

Admin stability improves with panic prevention and proper error propagation. Non-existent feature flags are removed from `lib.rs` documentation, and all previously undocumented `#[allow(...)]` attributes now carry explanatory comments.

## Breaking Changes

- ⚠️ [ModelAdmin: switch default permissions to deny-by-default](https://github.com/kent8192/reinhardt-web/discussions)
