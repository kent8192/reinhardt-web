# reinhardt-web v0.1.0-alpha.8

## Highlights

This release focuses on strengthening the project's development infrastructure and internal quality guardrails rather than end-user API changes.

The most notable addition is a Semgrep-based CI workflow that automatically detects unresolved `TODO`/`FIXME` comments and `todo!()` macro calls introduced in pull requests. Using diff-aware scanning, the check flags only newly added occurrences — leaving the existing backlog undisturbed — and blocks merges until they are resolved. Companion Clippy deny lints for `todo`, `unimplemented`, and `dbg_macro` reinforce the same policy at compile time. Together, these measures help prevent accidental technical debt from accumulating in the main branch.

On the documentation side, the release captures operational lessons learned while running release-plz in production: known pitfalls (such as circular publish dependencies and workspace version edge cases), recovery procedures for partial release failures, and configuration rationale are now documented in `RELEASE_PROCESS.md` and cross-referenced from `CLAUDE.md`.

## Breaking Changes

No breaking changes in this release.
