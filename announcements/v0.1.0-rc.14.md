# reinhardt-web v0.1.0-rc.14

## Highlights

This release introduces a composable settings system as its headline feature. The new `SettingsFragment` trait and `#[settings]` proc macro let applications compose only the settings they need — for example, `#[settings(CoreSettings | CacheSettings | SessionSettings)]` generates a typed `ProjectSettings` struct with full serde support, `Has*` accessor traits, and a `HasSettings<F>` blanket bridge that eliminates the need to import individual `Has*Settings` traits. Field-level Required/Optional policies, implicit field name inference, and TOML section-nested deserialization round out the system. The legacy `Settings` and `AdvancedSettings` structs are deprecated but remain available via backward-compatible `#[serde(flatten)]` bridges.

A critical runtime bug is fixed across every macro code path: the dependency injection context was not being forked per request in `#[get]`/`#[post]` route macros, `#[action]`, `#[server_fn]`, `#[grpc_handler]`, and `#[graphql_handler]`. Handlers relying on request-scoped services (such as `AuthInfo`) now resolve correctly. Alongside this, `#[server_fn]` gains Content-Type negotiation — HTML form submissions to JSON handlers are transparently converted — and `#[inject]` parameters are auto-detected without requiring `use_inject = true`.

Several security vulnerabilities are addressed: SQL injection vectors in PostgreSQL identifier quoting, ON CONFLICT clauses, batch insert, and MySQL user formatting are all closed; XSS via case-insensitive `</script>` variants and unsandboxed JS expression evaluation is hardened; session fixation is prevented by invalidating the prior session on login; path traversal sequences in `FileField` and `ImageField` are rejected; and distributed task locks gain atomic Redis acquisition with UUID-based ownership verification.

Broad correctness fixes land across the stack: Unicode string length validators now use `chars().count()` instead of byte length in validators, gRPC, GraphQL, and fuzzy search filters; pagination correctly reports total count and rejects zero `limit`; `BurstRateThrottle::wait_time` uses the correct prefixed cache keys; a WebSocket `InMemoryChannelLayer` ABBA deadlock is resolved; and the migration system fixes multi-element dependency parsing, squash dependency collection, and BFS topological sort ordering.

New capabilities include an `execute_from_command_line_with_registry()` entry point for custom management commands, fantoccini-based E2E browser testing fixtures in `reinhardt-test`, a declarative middleware exclusion API (`router.with_middleware(M).exclude("/path/")`), a `--index` CLI option for SPA `index.html` serving outside the static directory, and standalone boolean attribute support in the `page!` macro.

## Breaking Changes

No breaking changes in this release.
