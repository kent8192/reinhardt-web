# reinhardt-web v0.1.0-alpha.7

## Highlights

This release fixes a correctness issue in WebSocket authentication: `PagesAuthenticator::authenticate_from_cookies` previously contained a `todo!()` placeholder that would panic at runtime on certain error paths. It now returns a proper `AuthenticationFailed` error, giving callers predictable, handleable failure semantics. Error-case coverage for this method has also been added. Note that this is a behaviour-breaking change for any code that previously relied (intentionally or not) on the panic path.

The most significant infrastructure fix in this release resolves a circular publish-dependency deadlock that was preventing `cargo publish` from succeeding. Ten crates — including `reinhardt-db`, `reinhardt-auth`, `reinhardt-core`, `reinhardt-rest`, `reinhardt-views`, `reinhardt-di`, `reinhardt-http`, and `reinhardt-admin` — each listed `reinhardt-test` as a dev-dependency while `reinhardt-test` itself depended on those same crates, creating a resolution cycle under `cargo publish`. The cycle is broken per-crate using a combination of unused-dependency removal, local `poll_until` helper replacements, and test migration to a dedicated integration crate.

Two additional release-pipeline regressions are also addressed: a Cargo 1.84+ behaviour change that caused dev-dependency version resolution to fail against the crates.io index during packaging, and a `gix` slotmap overflow bug that caused partial publish failures. The CI workflow now clears the registry index cache before publishing, retries automatically on failure, and supports manual `workflow_dispatch` re-runs to recover from partial releases without requiring a new push to `main`.

## Breaking Changes

No breaking changes in this release.
