# reinhardt-web v0.1.0-rc.4

## Highlights

This release focuses on bug fixes that improve correctness across compilation targets, feature resolution, and database migrations.

Users targeting `wasm32` with the `full` feature enabled can now compile successfully. The `parallel` (rayon) and `jsonschema` validator modules lacked `not(target_arch = "wasm32")` platform gates, causing build failures on wasm32 targets. These gates are now in place, consistent with the pattern already used elsewhere in the codebase.

Two missing feature forwarding entries in the root `Cargo.toml` are corrected. The `redis-backend` feature now properly forwards to `reinhardt-utils/redis-backend`, restoring access to `RedisCache`, and the `middleware` feature now forwards to `reinhardt-auth/middleware`, restoring access to `HttpSessionConfig`, `SameSite`, and `SessionMiddleware`. These were inadvertently broken by a prior workspace dependency change.

A silent bug in the migration AST parser is fixed: the parser previously used `extract_static_str_field()` for name fields, which only matched bare string literals. Since framework-generated migration files use the `"name".to_string()` pattern, operations were silently skipped during parsing. The parser now uses `extract_string_field()` consistently across all operation types — `CreateTable`, `DropTable`, `AddColumn`, `DropColumn`, `AlterColumn`, `RenameTable`, `RenameColumn`, `CreateIndex`, `DropIndex`, and `ColumnDefinition` — handling both patterns correctly.

Additionally, `admin_routes` and `AdminRouter` are now re-exported from the `reinhardt::admin` namespace, making it possible to access admin routing from the top-level `reinhardt` crate without reaching into internal crate paths.

## Breaking Changes

No breaking changes in this release.
