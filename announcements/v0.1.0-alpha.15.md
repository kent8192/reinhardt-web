# reinhardt-web v0.1.0-alpha.15

## Highlights

This release introduces the new `reinhardt-deeplink` crate, adding core deep linking functionality to the Reinhardt ecosystem. The crate provides validation utilities and a comprehensive test suite, enabling URL-based navigation to specific content within Reinhardt applications and laying the groundwork for richer cross-app and in-app navigation patterns.

The examples directory receives a substantial overhaul, moving from a split `examples/local/` / `examples/remote/` layout to a single flat `examples/` structure. All examples now ship with dedicated settings files following a consistent pattern, and PostgreSQL-based examples include a `docker-compose.yml` for one-command database setup. A new `ci.toml` configuration with automatic CI environment detection makes it easier to run examples in continuous integration pipelines without manual configuration.

Developer experience for the examples is further improved with quick start instructions added to each example's `README.md`. Local development is supported through a simple `.cargo/config.toml` copy mechanism, replacing the previous workspace-member approach. Stale configuration files, orphaned settings from the old directory layout, and unused shared crates (`example-common`, `example-test-macros`) have all been removed, leaving a cleaner and more approachable workspace for contributors and users evaluating the framework.

## Breaking Changes

No breaking changes in this release.
