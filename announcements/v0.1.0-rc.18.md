# reinhardt-web v0.1.0-rc.18

## Highlights

This release resolves a cluster of hard-block bugs in the `reinhardt-admin startproject` and `startapp` commands. Freshly scaffolded pages projects previously failed to compile for several reasons: the `[[bin]]` entry used `{{ project_name }}` instead of the required `"manage"` literal; the `#[settings(/* ... */)]` placeholder stripped to an invalid empty attribute at parse time; and `ws_urls.rs` was never generated, causing an E0433 resolver error. The hidden-file filter in the template processor also silently discarded `.gitignore.tpl`, meaning `startproject` has never emitted a `.gitignore` — a security concern since `local.toml` credentials would not be gitignored. All of these are now fixed. Generated `local.example.toml` files use fully-commented stubs so no active secrets appear by default, and the pages router template now uses the correct `reinhardt::pages::router::Router` API instead of raw `wasm-bindgen`/`web-sys` bindings.

The `#[server_fn]` macro now accepts `FromRequest`-based extractors (`Validated`, `Json`, `Form`, `Header`, `Cookie`, `Path`, `Query`, `Body`, etc.) as first-class parameters. The macro resolves them via `FromRequest::from_request` on the server side and excludes them from the WASM client's argument struct — reducing boilerplate and bringing `#[server_fn]` in line with how `#[view]` handlers work.

A new `reinhardt_pages::ClientLauncher` builder encapsulates all WASM startup boilerplate — panic hook, reactive scheduler, DOM mounting, history listener — behind a single `.launch()` call, eliminating ~80 lines of repeated setup in every new pages project. Alongside this, `ClientRouter::render_current()` now returns `Page` (falling back to `Page::Empty`) instead of `Option<Page>`, which better reflects its always-available semantics.

The `reinhardt-conf` crate now emits an actionable warning when flat TOML keys appear outside the `[core]` section, preventing silent configuration drops that were previously impossible to diagnose without reading source code. Extensive documentation has also been aligned with the current implementation, including replacing all references to the removed `--template-type` CLI flag and correcting stale version pins throughout tutorials and examples.
